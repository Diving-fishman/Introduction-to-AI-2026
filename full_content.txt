## 一、📜 人工智能发展简史

### 知识点 1：AI 是什么时候诞生的？

**1956 年，达特茅斯会议（Dartmouth Workshop）** 🏛️

- 发起人：**John McCarthy、Marvin Minsky、Claude Shannon** 等 10 位科学家
- 这是**人工智能**这个学科正式诞生的标志 🎂
- 最初定义："让机器做出需要人类智能的行为"

> 💡 达特茅斯会议只开了 2 个月，但影响了之后 60 多年的科技走向！

### 知识点 2：AI 发展的三次浪潮 🌊

#### 第一次浪潮（1956-1969）🚀

**成就：**
- 1960 年 McCarthy 发明 **LISP 语言**
- 1965 年 **DENDRAL** 专家系统（化学分子结构分析）
- 1969 年 **IJCAI** 国际人工智能联合会议成立

**代表人物：**
- **McCarthy**（LISP 之父，AI 先驱）
- **Minsky**（MIT 实验室，框架理论）
- **Newell & Simon**（逻辑理论家、通用问题求解器 GPS）

#### 第一次寒冬（1970s）❄️

**原因：**
- 1966-1974 年机器翻译项目失败
- Lighthill Report（1973）对 AI 提出严厉批评
- 算力不足、算法有限

#### 第二次浪潮（1980s）🎯

**成就：**
- **专家系统**大放异彩（1977 年 Feigenbaum 提出"知识工程"）
- MYCIN 医疗诊断系统、XCON 配置系统
- **统计学方法**开始崛起（Bayes 网络）
- 1986 年 BP 算法

**特点：** 从"算法为王"转向"知识为王"

> 💡 专家系统的 slogan："知识就是力量" 💪

#### 第二次寒冬（1990s）❄️

**原因：**
- 专家系统维护成本高、难以扩展
- 日本五代机项目失败
- 神经网络再次进入低谷（SVM 崛起）

#### 第三次浪潮（2011-至今）💥

**标志事件：**
- 2012 **AlexNet** ImageNet 夺冠 → 深度学习爆发
- 2016 **AlphaGo** 击败李世石 🏆
- 2017 Transformer → ChatGPT → 大模型时代
- 计算机视觉、自然语言处理、语音识别全面突破

**驱动力：**
- ✅ **大数据**（互联网时代的海量数据）
- ✅ **大算力**（GPU/TPU）
- ✅ **好算法**（深度学习、强化学习）

### 知识点 6：图灵测试（Turing Test, 1950）🧪

**核心思想**：如果一台机器在对话中让人无法分辨它是人还是机器，就说明它有智能。

> 2014 年有程序声称通过了图灵测试，但争议很大。现在看图灵测试只是必要条件，不是充分条件。

**图灵测试的局限性**：
- 只测试"模仿人类"的能力，不测试真正的智能
- "中文房间"思想实验 🏠：就算能完美回答中文问题，也不代表理解中文

# 🔍 搜索 复习笔记

> 一句话说清搜索：**在一个"问题空间"里一步步试探，最终找到通往目标的路径。** 🗺️ → 🎯

## 📌 知识地图（先看整体框架）

```text
                   搜索问题
                      │
        ┌─────────────┴─────────────┐
        │                            │
   问题形式化                   搜索策略
  （状态、动作、目标）               │
                          ┌─────────┴─────────┐
                          │                    │
                    无信息搜索             有信息搜索
                    (盲搜)                 (启发式)
                          │                    │
                    ┌─────┼─────┐              │
                    │     │     │              │

                                              │
                                          A* 最优性条件
                                        (可采纳/一致性)
```

## 一、❓ 什么是搜索？

### 知识点 1：AI 为什么需要搜索？

AI 系统可以分为三大类：

| 类型 | 方法 | 例子 |
| :---- | :---- | :---- |
| **基于算法（ABS）** 🧮 | 数学建模、推公式 | 线性规划、优化 |
| **基于搜索（SBS）** 🔍 | 在解空间中搜索 | **AlphaGo/AlphaZero**、路径规划 |
| **基于案例（CBS）** 📚 | 找相似历史经验 | 医疗诊断、推荐系统 |

> 💡 **SBS 的特点**：不需要理解问题本身，只需要能"试"和"判断好坏"。围棋有 10¹⁷² 种状态，比宇宙原子数还多 🌌，但 AlphaGo 靠搜索就能找到最优走法。

### 知识点 2：搜索问题的形式化

一个搜索问题需要定义以下 5 个要素：

| 要素 | 符号 | 含义 | 例子：在罗马尼亚旅行 🚗 |
| :---- | :---- | :---- | :---------------------- |
| **状态空间** 🗺️ | S | 所有可能的状态 | 各个城市 |
| **初始状态** 🚩 | s₀ | 开始的位置 | Arad |
| **动作集合** 🎮 | A(s) | 在状态 s 可以做哪些操作 | 开往相邻城市 |
| **转移函数** 🔄 | Result(s,a) | 做了 a 后的新状态 | Arad→Sibiu |
| **目标测试** 🎯 | G(s) | 判断是否到达目标 | 到 Bucharest 了吗？ |
| **路径代价** 💰 | c(s,a,s') | 走这一步的代价 | 道路距离（km） |

> 搜索的本质：从初始状态出发，通过一系列动作转移到目标状态 🚩 → ... → 🎯

### 知识点 3：状态空间图 vs 搜索树

```text
状态空间图（State Space Graph）🌐        搜索树（Search Tree）🌳
                                      
    S ──→ a ──→ G                      S
    │      │      │                    ┊
    │      │      │                 ┌──┴──┐
    ↓      ↓      ↓                 a     d ...
    b ←─── c ←─── e                 ┊
    │      │      │              ┌──┴──┐
    ↓      ↓      ↓              b     a ...
    d ──→ p ──→ q
```

| | 状态空间图 | 搜索树 |
| :-- | :---------- | :------ |
| 节点 | 所有可能的**状态** | 从初始状态出发的**路径** |
| 包含 | 整个问题空间 | 探索过的部分 |
| 重复节点 | 每个状态出现一次 | 可能多次出现（路径不同） |

## 二、🔎 搜索算法的评估维度

### 知识点 4：评估搜索算法的四个标准

| 标准 | 含义 | 问的问题 |
| :---- | :---- | :-------- |
| **完备性** ✅ | 保证能找到解吗？ | 只要存在答案，能找到吗？ |
| **最优性** 🏆 | 找到的解是最优的吗？ | 找到的路径是最短的吗？ |
| **时间复杂度** ⏱️ | 运行多快？ | 要探索多少个节点？ |
| **空间复杂度** 💾 | 需要多少内存？ | 要存多少个节点？ |

---

## 三、🕶️ 无信息搜索（Uninformed Search / Blind Search）

> 核心特点：除了问题定义本身，没有任何额外信息。就像蒙着眼睛找路 😵

### 知识点 5：深度优先搜索（DFS）⬇️

**核心思路**：一条路走到黑，走不通再回头。

```text
DFS 就像走迷宫 🌀：
选一条路一直走 → 撞墙了 → 退回上一个岔路口 → 换一条路再走
```

**实现方式**：**栈（Stack，FILO）**
- OPEN 表（待探索节点）：后进先出
- CLOSED 表（已探索节点）

| 评估 | 结果 |
| :---- | :---- |
| 完备性 | ❌ 可能陷入无限深分支 |
| 最优性 | ❌ 不一定找到最短路径 |
| 时间复杂度 | O(b^m)（b=分支因子，m=最大深度） |
| 空间复杂度 | O(bm) |

### 知识点 6：广度优先搜索（BFS）➡️

**核心思路**：一层一层推进，同一层的节点全部探索完再进入下一层。

```text
BFS 就像水波扩散 🌊：
从起点开始 → 先看一步能到哪 → 再看两步能到哪 → 逐层扩大
```

**实现方式**：**队列（Queue，FIFO）**
- OPEN 表：先进先出

| 评估 | 结果 |
| :---- | :---- |
| 完备性 | ✅（如果分支因子有限） |
| 最优性 | ✅（如果每步代价相同） |
| 时间复杂度 | O(b^d)（d=解的深度） |
| 空间复杂度 | **O(b^d)**（所有节点都要存 😱） |

> 💡 BFS 的最大问题：**空间爆炸**。每层的节点数指数增长，很快就会存不下。

### 知识点 7：代价一致搜索（UCS）💰

**核心思路**：优先扩展**累计代价最小**的节点，而不是深度最浅的。

```text
UCS = BFS 的升级版 🚀
BFS 默认每步代价=1，UCS 考虑不同动作的代价不同

类比：去一个地方有好几条路 🛣️
BFS：换乘次数最少的路线
UCS：总路程最短的路线
```

**实现方式**：**优先队列（Priority Queue）**，按 g(n) 排序

- g(n)：从起点到节点 n 的实际代价
- 每次从 OPEN 表中取出 g(n) 最小的节点扩展

```text
UCS 的代价等高线（Cost Contours）：

      ┌─── C=1 ─── ─── C=2 ─── ─── C=3 ─── ...
      │              │              │
      S ──C=3──→ d   e ──C=2──→ ...
      │              │
      └─── C=1──→ p ──C=15──→ q ...

UCS 按代价分层扩展，先探索代价小的区域
```

| 评估 | 结果 |
| :---- | :---- |
| 完备性 | ✅（如果每步代价>0） |
| 最优性 | ✅ |
| 时间复杂度 | O(b^(1+⌊C*/ε⌋)) |
| 空间复杂度 | O(b^(1+⌊C*/ε⌋)) |

---

## 四、💡 有信息搜索（Informed Search / Heuristic Search）

> 核心特点：利用启发式信息（额外的知识）指导搜索方向，避免盲目乱走 🧭

### 知识点 8：启发式函数（Heuristic Function）

**h(n)** = 从节点 n 到**目标**的**估计**代价 🎯

> 类比：你在陌生城市找目的地 🗺️
> - g(n) = 你已经走了多远（已知）
> - h(n) = 你**估计**还有多远（启发式——可能不准）

**评价函数 f(n) = g(n) + h(n)**
- g(n) = 从起点到 n 的实际代价（已知道路）
- h(n) = 从 n 到目标的估计代价（启发式——需设计）

### 知识点 9：最佳优先搜索（Best-First Search）

| 变体 | 评价函数 | 行为 |
| :---- | :-------- | :---- |
| **贪心最佳优先** 🤑 | f(n) = h(n) | 只看"离目标多近"，不顾已走距离 |
| **UCS** 💰 | f(n) = g(n) | 只看"已走距离"，不顾离目标多远 |
| **A\*** ⭐ | f(n) = g(n) + h(n) | **兼顾两者**，最聪明 |

### 知识点 10：A\* 算法（重点 ⭐）

**A\* 是搜索算法的"天花板"** 🏆

**核心思想**：把"已经付出的代价"和"未来预计的代价"加起来，选总和最小的节点。

```text
f(n) = g(n) + h(n)

g(n) = 从起点到 n 的实际代价    ← 过去
h(n) = 从 n 到目标的估计代价   ← 未来（启发式）
f(n) = 经过 n 的路径的总估计代价
```

**A\* 的 OPEN 表**：按 f(n) 值排序，每次取出 f 最小的节点扩展。

**A\* 求解八数码问题示例：**

```text
初始状态 S0            目标状态
 2  8  3               1  2  3
 1  6  4        →      8  0  4
 7  0  5               7  6  5

h(n) = 错位数（不在正确位置的数字个数）
g(n) = 已走步数
f(n) = g(n) + h(n)

搜索过程：
S0: g=0, h=5, f=5 → 扩展
├─ A: g=1, h=6, f=7
├─ B: g=1, h=6, f=7
├─ C: g=1, h=4, f=5 → f 最小，先扩展
│   ├─ D: g=2, h=5, f=7
│   ├─ E: g=2, h=4, f=6
│   │   ...
│   └─ K: g=5, h=0, f=5 → 到达目标！🎉
```

### 知识点 11：A\* 的最优性条件（必考 ⭐）

#### 条件一：可采纳性（Admissibility）

**h(n) ≤ h\*(n)** — 启发式函数**不高于**真实代价

> 💡 **h(n) 必须"乐观估计"**，不能高估到目标的距离。
> 比如到目标实际最少还需 10km，h(n) 最大只能是 10，不能是 15。
>
> **结论**：如果 h(n) 是可采纳的，A\* 一定能找到最优解 ✅

#### 条件二：一致性 / 单调性（Consistency / Monotonicity）

**h(n) ≤ c(n,a,n') + h(n')**

> 从 n 到目标的估计代价，不能大于"走一步的实际代价 + 下一步的估计代价"。
>
> **结论**：如果 h(n) 是一致的，A\* 在第一次扩展节点 n 时就找到了到 n 的最优路径。

### 知识点 12：设计更好的启发式

**两个启发式函数的比较：**
```text
h₁(n) = 错误放置的棋子数（简单）
h₂(n) = 每个棋子到目标位置的曼哈顿距离之和（更精确）

对同一个问题：h₂(n) ≥ h₁(n)，因此 h₂ 更好
```

> 💡 **h(n) 越大越好**（但不能超过 h\*(n)）。更精确的启发式意味着更少的搜索节点。

**Dominance（优势关系）**：如果 h₂(n) ≥ h₁(n) 对所有 n 成立，那么 h₂ **支配** h₁，用 h₂ 搜索的节点数 ≤ 用 h₁ 的。

---

## 五、🧩 经典搜索问题

### 知识点 13：TSP（旅行商问题）🌍

**问题**：给定 n 个城市和城市间的距离，找一条最短路径，**每个城市恰好走一次**再回到起点。

```text
例子：5 个城市 A、B、C、D、E

所有可能路径数 = (n-1)! / 2
当 n=5 时: 12 种
当 n=30 时: 4.9×10²³ 种 → 穷举需要远超宇宙寿命 💀
```

TSP 是 **NP-hard** 问题，大规模时不可能在合理时间内找到最优解。实际中用启发式方法找"足够好"的解。

### 知识点 14：背包问题 🎒

**问题**：给定 N 个物品（每个有重量和价格），背包容量有限，怎么装最值钱？

```text
物品:  重量(kg)  价值($)
1      4         12        ← 性价比 3
2      2         1         ← 性价比 0.5
3      10        4         ← 性价比 0.4
4      1         1         ← 性价比 1
5      2         2         ← 性价比 1

背包容量：16 kg
搜索空间：2⁵ = 32 种组合
```

### 知识点 15：Dijkstra 最短路径算法 🛣️

**Dijkstra 算法 = A\* 的特殊情况（h(n) = 0）**

| | Dijkstra | A\* |
| :-- | :--------- | :--- |
| 评价函数 | f(n) = g(n) | f(n) = g(n) + h(n) |
| 方向性 | 从起点向所有方向均匀扩展 | 朝着目标方向扩展 |
| 效率 | 慢（扩展太多节点） | 快（有方向感） |

Dijkstra 算法步骤：
1. 起点距离 = 0，其他节点 = ∞
2. 取出当前距离最小的节点
3. 更新相邻节点的距离（松弛操作）
4. 重复直到所有节点被处理

# 📊 机器学习 复习笔记

> 一句话说清机器学习：**让计算机从数据中自动学习规律，然后用学到的规律做预测或决策。** 📈 → 🧠 → 🎯

## 📌 知识地图（先看整体框架）

```text
                     机器学习
                         │
        ┌────────────────┼────────────────┐
        │                │                │
    监督学习          无监督学习        强化学习
   （有标签）         （无标签）        （奖励信号）
        │                │                │
   ┌────┴───┐       ┌────┴────┐           │
   │        │       │         │           │
  分类     回归    聚类      降维        Q-Learning等
（离散值）（连续值）（K-Means）（PCA）
        │
   ┌────┴──────────────┐
   │                   │
  弱监督学习         迁移学习
   ┌──┼──┐             │
  不完全  不确切  不准  预训练+微调
```

## 一、🧠 机器学习基础概念

### 知识点 1：什么是机器学习？

**定义**：计算机利用**经验**（数据）来改善**性能**指标的过程。

```
输入：数据（经验）
过程：学习算法从数据中发现规律
输出：一个模型（可以用来做预测）
```

**一个核心公式**：
```text
学习算法 + 训练数据 → 模型 → 预测新数据
```

### 知识点 2：基本术语

| 术语 | 英文 | 含义 | 比喻 🌰 |
| :---- | :---- | :---- | :------ |
| **样本** | Sample / Instance | 一条数据记录 | 一个学生的信息 |
| **特征** | Feature | 样本的某个属性 | 年龄、身高、成绩 |
| **特征向量** | Feature Vector | 所有特征组成的向量 | (年龄, 身高, 成绩) |
| **标签** | Label | 我们要预测的目标值 | 是否被录取 ✅❌ |
| **数据集** | Data Set | 所有样本的集合 | 全班学生的数据 |
| **训练集** | Training Set | 用来训练模型的数据 | 过去学生的数据 |
| **测试集** | Test Set | 用来评估模型的数据 | 新学生的数据 |

### 知识点 3：几个重要的假设

| 假设 | 含义 |
| :---- | :---- |
| **独立同分布（IID）** 📊 | 训练数据和测试数据来自同一分布，且相互独立 |
| **数据充分性** ✅ | 数据量要足够大，才能学到可靠规律 |

---

## 二、📐 机器学习的核心框架

### 知识点 4：模型、假设空间与学习

**模型** = 从输入到输出的映射函数：

```text
线性回归：y = wx + b              ← 参数 w 和 b 控制模型行为
神经网络：y = σ(W·x + b)          ← 参数 W 和 b
决策树：一系列 if-then 规则        ← 树结构参数
```

**假设空间（Hypothesis Space）**：所有可能的模型组成的集合。

```text
假设空间 H = { f(x; θ) | θ ∈ Θ }

θ 是模型参数，Θ 是所有可能的参数取值
H 中包含了所有可能的学习结果
```

> 💡 机器学习 = 在假设空间中搜索一个"最好"的模型，使其在训练数据上表现最好。

### 知识点 5：损失函数与风险

**损失函数 L(y, f(x))**：衡量模型预测值和真实值之间的差距。

| 任务 | 常用损失函数 |
| :---- | :---- |
| **回归** 📈 | 均方误差 MSE：L(y, ŷ) = (y - ŷ)² |
| **分类** 🏷️ | 交叉熵（Cross-Entropy）、0-1 损失 |

**三种风险最小化：**

```text
① 期望风险（Expected Risk）—— 理论上最优，但算不了
   R(θ) = ∫ L(y, f(x,θ)) · p(x,y) dx dy

② 经验风险（Empirical Risk）—— 用训练数据近似
   R_emp(θ) = 1/N · Σ L(yᵢ, f(xᵢ,θ))

③ 结构风险（Structural Risk）—— 加正则项防过拟合
   R_srm(θ) = R_emp(θ) + λ·Ω(θ)
```

### 知识点 6：过拟合 vs 欠拟合

```text
      欠拟合（Underfitting）         过拟合（Overfitting）
           😅                             😰
    ┌──────────────────┐       ┌──────────────────┐
    │  模型太简单       │       │  模型太复杂       │
    │  训练集也学不好    │       │  训练集完美但      │
    │  测试集更差       │       │  测试集很差        │
    └──────────────────┘       └──────────────────┘

           👍 刚刚好（Good Fit）
    ┌──────────────────┐
    │  模型复杂度适中    │
    │  训练集和测试集    │
    │  表现都很好        │
    └──────────────────┘
```

**训练过程示意图：**
```text
误差
↑
│     训练误差
│    ┌──┐
│   ┌┘  └──┐            测试误差
│  ┌┘      └──┐       ┌──┐
│ ┌┘          └──┐   ┌┘  └──┐
│ ┌┘              └───┘      └──────→ 模型复杂度
│←--- 欠拟合 ---→←--- 过拟合 ---→
```

| 问题 | 原因 | 解决方法 |
| :---- | :---- | :---- |
| **欠拟合** 😅 | 模型太简单 / 特征不够 | 增加模型复杂度、增加特征 |
| **过拟合** 😰 | 模型太复杂 / 数据太少 | 正则化、减少特征、增加数据 |

### 知识点 7：机器学习的典型流程

```text
① 收集数据 📝
② 数据预处理（清洗、归一化、特征工程） 🧹
③ 划分训练集 / 验证集 / 测试集 ✂️
④ 选择模型（线性回归、SVM、神经网络等）
⑤ 设置超参数（学习率、正则化系数等） ⚙️
⑥ 在训练集上训练模型 🏋️
⑦ 在验证集上调优 🔧
⑧ 在测试集上评估 🧪
⑨ 部署到实际应用 🚀
```

## 三、🏷️ 监督学习（Supervised Learning）

### 知识点 8：什么是监督学习？

**输入**：带标签的训练数据 {(x₁, y₁), (x₂, y₂), ..., (xₙ, yₙ)}
**任务**：学习从 x 到 y 的映射 f: X → Y

```text
             ┌── 分类（Classification）
监督学习 ────┤                y 是离散的（猫/狗、正/负）
             │
             └── 回归（Regression）
                          y 是连续的（房价、温度）
```

**常见算法：**
| 算法 | 类型 | 一句话说清 |
| :---- | :---- | :--------- |
| **线性回归** 📈 | 回归 | y = wx + b，找到最拟合数据的直线 |
| **逻辑回归** 📉 | 分类 | 用 Sigmoid 函数把输出变成 [0,1] 的**概率** |
| **决策树** 🌳 | 分类/回归 | 一堆 if-then 规则组成的树 |
| **SVM** 🏋️ | 分类 | 找到两个类别之间最宽的"间隔" |
| **朴素贝叶斯** 🧮 | 分类 | 用概率公式 P(y\|x) ∝ P(x\|y)P(y) |

## 四、🔍 无监督学习（Unsupervised Learning）

### 知识点 9：什么是无监督学习？

**输入**：只有数据 xᵢ，没有标签 yᵢ
**任务**：发现数据中隐藏的结构或模式

```text
               ┌── 聚类（Clustering）— 物以类聚
无监督学习 ────┤
               └── 降维（Dimensionality Reduction）— 简化数据
```

### 知识点 10：K-Means 聚类

**核心思想**：把数据分成 K 个簇，让**同一簇内的点尽量近，不同簇的点尽量远**。

**K-Means 的步骤：**
```text
① 随机选 K 个点作为初始"中心"
② 每个点分给离它最近的中心
③ 更新每个簇的中心（取簇内点的平均值）
④ 重复 ②③ 直到中心不再变化
```

```text
初始状态        第1轮        第3轮        收敛
  · · ·        · · ·        · · ·        · · ·
  · · ·   →    · · ·   →    · · ·   →    · · ·
 ·⊕· ·· ⊕·    ·⊕· ·· ⊕·    ·⊕· ·· ⊕·    ·⊕· ·· ⊕·
  · · ·        · · ·        · · ·        · · ·
  · · ·        · · ·        · · ·        · · ·
⊕ = 簇中心    · = 数据点    颜色 = 隶属簇
```

**常见降维方法**：**PCA**（主成分分析）📉

## 五、🔄 其他学习范式

### 知识点 11：弱监督学习（Weakly Supervised Learning）

**定义**：使用不完美监督信号的学习方法。包含三种情况：

| 类型 | 说明 | 例子 |
| :---- | :---- | :---- |
| **不完全监督** 🕳️ | 只有部分数据有标签 | 100 万张图，只有 1000 张有标签 |
| **不确切监督** ❓ | 标签不够精确 | 一张图上有猫和狗，但只标了"动物" |
| **不准确监督** ⚠️ | 标签可能有错误 | 标注人员标错了类别 |

### 知识点 12：迁移学习（Transfer Learning）🎁

**核心思想**：把在某个任务上学到的知识，应用到另一个相关任务上。

```text
传统学习：对每个新任务都要从头学起 ❌
        任务A → 模型A    任务B → 从头训练模型B

迁移学习：在任务A上学到知识，迁移到任务B ✅
        任务A → 基础模型 → 微调 → 任务B的模型
```

> 💡 **例子**：用 ImageNet（1000 类图片分类）训练好的 CNN，拿去识别医学影像（只需要少量医学图片微调）。

### 知识点 13：更多学习范式一览

| 范式 | 一句话说清 |
| :---- | :--------- |
| **自监督学习** 🪞 | 从数据本身构造监督信号（如 BERT 预测被遮住的词） |
| **度量学习** 📏 | 学一个距离函数，让同类样本距离近，异类远 |
| **对比学习** 🔁 | 让相似样本的表示靠近，不相似的远离 |
| **对抗学习** 🎭 | 生成对抗样本攻击模型，再让模型学会抵抗 |
| **元学习** 🧠 | "学会学习"——在新任务上快速适应 |

## 六、📚 学习理论要点

### 知识点 14：没有免费午餐定理（No Free Lunch Theorem）🍽️

> **核心结论**：没有一种算法在所有问题上都是最好的。

```text
算法A在问题集P₁上表现好
算法B在问题集P₂上表现好

如果对所有可能的问题求平均，所有算法的期望性能是一样的。
```

**实际意义**：选择算法需要根据**具体问题**判断，不能盲目迷信某种算法。

### 知识点 15：丑小鸭定理（Ugly Duckling Theorem）🦆

> **核心结论**：没有绝对客观的分类标准，分类依赖于我们的**主观偏好**。

```text
丑小鸭和白天鹅的"差异"，取决于你用什么标准来衡量：
- 按外形分：丑小鸭 ≠ 白天鹅
- 按基因分：丑小鸭 = 白天鹅（都是天鹅科）
```

### 知识点 16：奥卡姆剃刀（Occam's Razor）🪒

**Entities should not be multiplied unnecessarily.** —— William of Occam

> **核心结论**：在效果相同时，**选择最简单的那个模型**。

```text
如果有两个模型在训练集上表现一样好：
  模型A：3 个参数（简单） 👍
  模型B：30 个参数（复杂）

奥卡姆剃刀说：选模型A！更简单的模型更可能泛化到新数据。
```

### 知识点 17：归纳偏置（Inductive Bias）🎯

> **核心问题**：数据是有限的，但可能的模型是无限的。如何选择？

**归纳偏置** = 学习算法在假设空间中做选择时的**先验偏好**。

| 算法 | 归纳偏置 |
| :---- | :---- |
| **线性回归** 📈 | 假设 x 和 y 的关系是线性的 |
| **CNN** 🧠 | 假设图像特征具有局部性和平移不变性 |
| **RNN** 🔄 | 假设数据具有时序依赖 |
| **决策树** 🌳 | 倾向于选择较短的树 |

> 💡 **没有归纳偏置就无法学习**——数据本身不足以确定唯一模型，必须加上先验假设。

# 🤔 符号学派 复习笔记

> 一句话说清符号主义：**智能 = 符号操作。把知识写成符号公式，用逻辑规则推导结论。** 📝 → 🔗 → 💡

```text
                     符号学派
                         │
        ┌────────────────┼────────────────┐
        │                │                │
   逻辑推理         知识表示         不确定性推理
        │                │                │
```

## 一、📜 什么是符号学派？

### 知识点 1：符号学派的核心理念

**智能的本质 = 对符号进行操作和推理** 🧮

```text
符号主义的信念：
  ✅ 知识可以用"符号"来表示（如"猫"、"动物"）
  ✅ 推理就是按照规则操作这些符号
  ✅ 只要知识库足够大、推理规则足够强，就能实现智能

典型代表：
  · "知识就是力量" —— 专家系统时代的口号 💪
  · "物理符号系统假说" —— Newell & Simon
```

**符号学派的六步框架：**
```text
① 问题建模    ② 定义符号    ③ 构建规则
④ 推理求解    ⑤ 解释结果    ⑥ 知识更新
```

### 知识点 2：符号学派的优势与局限

| 优势 ✅ | 局限 ❌ |
| :------ | :------ |
| 推理过程**可解释**（能说清为什么） | 知识获取困难（人工输入规则成本高） |
| 可以处理**小样本**问题 | 难以处理**不确定性** |
| 知识可以**积累和复用** | 规则的"组合爆炸" 💥 |
| 逻辑严密，**结果确定** | 面对开放世界非常脆弱 |

## 二、🔗 知识表示（Knowledge Representation）

### 知识点 3：知识表示的基本概念

**知识表示 = 把现实世界的信息编码成计算机可以理解和操作的形式**

```text
知识表示的四种方法：
① 逻辑表示法（谓词逻辑） 📋
② 产生式表示法（IF-THEN 规则） ➡️
③ 框架表示法（对象-属性） 🖼️
④ 语义网络 / 知识图谱 🕸️
```

### 知识点 4：知识表示的理想特性

| 特性 | 含义 |
| :---- | :---- |
| **充分表示** 🎯 | 能表达所有需要表示的知识 |
| **充分推理** 🧠 | 支持对表示的知识进行推理 |
| **充分高效** ⚡ | 存储和操作效率高 |
| **充分获取** 📥 | 方便知识获取和更新 |

### 知识点 5：产生式系统（Production System）—— IF-THEN 规则

**最经典的符号主义知识表示方法**：

```text
IF <条件> THEN <结论/动作>

例子：
  IF 有发热 AND 有咳嗽 THEN 可能是感冒
  IF 是动物 AND 会飞 AND 有羽毛 THEN 是鸟
```

**产生式系统的三要素：**

| 要素 | 作用 | 比喻 📚 |
| :---- | :---- | :------ |
| **规则库** 📋 | 存储所有 IF-THEN 规则 | 法律条文汇编 |
| **综合数据库** 📦 | 存储当前已知的事实 | 案件事实记录 |
| **推理机** ⚙️ | 选择并执行规则 | 法官判案 |

**推理机的工作方式：**
```text
① 匹配：找所有条件满足当前事实的规则
② 冲突消解：如果多条规则同时满足，选哪条？
   - 优先级最高
   - 条件最具体
   - 最近用过的
③ 执行：应用选中的规则，更新事实库
④ 重复直到得出结论
```

**一个专家系统的推理过程：**
```text
事实：感冒了、发热、咳嗽
  ↓
匹配规则：IF 发热 AND 咳嗽 THEN 可能是流感
  ↓
结论：可能是流感 → 建议去医院检查
```

## 三、🧮 逻辑（Logic）—— 符号推理的基础

### 知识点 6：什么是逻辑？

**逻辑 = 研究正确推理形式的科学**

```text
"1+1=2" vs "1+1=10" 
两种说法都对，但基于不同的"世界"（十进制 vs 二进制）
逻辑就是告诉我们：在给定前提下，什么结论是必然成立的。
```

### 知识点 7：三种推理方式

| 推理 | 方向 | 特点 | 例子 |
| :---- | :---- | :---- | :---- |
| **演绎推理** 🔽 | 一般 → 个别 | 结论必然成立 | 所有人会死，苏格拉底是人 → 苏格拉底会死 |
| **归纳推理** 🔼 | 个别 → 一般 | 结论不一定成立 | 看到 100 只天鹅都是白的 → 天鹅都是白的？❌ |
| **默认推理** ➡️ | 缺省推理 | 没有反例就默认成立 | 鸟会飞 → 但企鹅不会飞（例外） |

### 知识点 8：命题逻辑（Propositional Logic）

**命题 = 可以判断真假的陈述句。**

```text
  P: "今天是晴天"  → 真或假
  Q: "我去游泳"    → 真或假
  
  逻辑连接词：
  ¬P   (非)    "今天不是晴天"
  P∧Q  (与)    "今天是晴天 且 我去游泳"
  P∨Q  (或)    "今天是晴天 或 我去游泳"
  P→Q  (蕴含)  "如果今天是晴天，我就去游泳"
  P↔Q  (等价)  "今天是晴天当且仅当我去游泳"
```

**真值表：**
```text
P    Q    ¬P    P∧Q    P∨Q    P→Q    P↔Q
T    T    F     T      T      T      T
T    F    F     F      T      F      F
F    T    T     F      T      T      F
F    F    T     F      F      T      T
```

> ⚠️ **P→Q 的重要规则**：只有当 P 真且 Q 假时，P→Q 为假。P 假时 P→Q 恒真！

### 知识点 9：谓词逻辑（Predicate Logic / FOL）—— 重点 ⭐

**命题逻辑的局限**：无法表达"所有人都会死"这种包含**量词**和**变量**的陈述。
**谓词逻辑 = 命题逻辑的升级版** 🚀

**核心元素：**

```text
① 谓词（Predicate）：表示关系或属性
   "x > y" → Greater(x, y)
   "x是人" → Human(x)
   "x喜欢y" → Like(x, y)

② 量词（Quantifier）：
   ∀x（全称量词）：所有 x 都...
     ∀x Human(x) → Mortal(x)   "所有人都难免一死"
   ∃x（存在量词）：存在某个 x 使得...
     ∃x Human(x) ∧ Smart(x)   "存在聪明的人"

③ 个体 / 常量：
   Socrates, Lisi, 华科...

④ 函数：
   son(x)  "x的儿子"
   father(x) "x的父亲"
```

**用谓词逻辑表达知识：**

```text
"所有学生都学习过人工智能课程"
∀x Student(x) → Study(x, AI)

"有的学生喜欢所有教授"
∃x Student(x) ∧ ∀y Professor(y) → Like(x, y)

"没有免费的午餐"
¬∃x Free(x) ∧ Lunch(x)
```

### 知识点 10：谓词逻辑推理示例

**经典三段论在谓词逻辑中的表达：**

```text
前提1：所有人都难免一死
        ∀x Human(x) → Mortal(x)
前提2：苏格拉底是人
        Human(Socrates)
结论：苏格拉底难免一死
        Mortal(Socrates)

推理过程：
① ∀x Human(x) → Mortal(x)
   用 Socrates 替换 x → Human(Socrates) → Mortal(Socrates)
② Human(Socrates) 已知为真
③ 所以 Mortal(Socrates) 为真 ✅
```

**更复杂的例子——"王喜欢聪明的东西"：**

```text
已知：
① ∀x Easy(x) → Like(Wang, x)    "王喜欢容易的科目"
② ∀x C(x) → Easy(x)              "所有计算机课都是容易的"
③ C(AI)                            "AI 是计算机课"

证明：Like(Wang, AI)  "王喜欢AI"

推理：
④ C(AI) → Easy(AI)              从②代入 x=AI
⑤ Easy(AI)                       从③④用假言推理
⑥ Easy(AI) → Like(Wang, AI)     从①代入 x=AI
⑦ Like(Wang, AI) ✅              从⑤⑥用假言推理
```

### 知识点 11：谓词逻辑的翻译练习（必做题 ⭐）

| 中文 | 谓词逻辑表达 |
| :---- | :---- |
| 人人都有母亲 | ∀x ∃y Mother(y, x) |
| 不存在既穿红又穿绿的人 | ¬∃x Red(x) ∧ Green(x) |
| 每个正整数都有唯一一个后继 | ∀x ∃y Succ(y, x) ... |

## 四、⚖️ 不确定性推理

### 知识点 12：为什么需要处理不确定性？

现实世界中的知识往往**不是确定的**：

```text
"发烧 + 咳嗽 → 可能是感冒"  （不是 100% 确定）
"乌云密布 → 可能要下雨"     （不是 100% 确定）
```

**不确定性的来源：**
- 知识本身不完整 🕳️
- 证据不充分 ❓
- 规则本身是统计性的 📊

### 知识点 13：确定性因子模型（C-F Model）—— 重点 ⭐

**MYCIN 医疗专家系统（1975, Shortliffe）** 使用的方法。

#### 基本概念

**CF（Certainty Factor，确定性因子）**：表示结论的可信程度，取值范围 [-1, 1]

```text
CF(H,E) = 可信度
├─ CF > 0：证据支持结论（越大越肯定）
├─ CF < 0：证据反对结论（越小越否定）
└─ CF = 0：证据与结论无关

例子：
  IF 发热 AND 咳嗽 THEN 感冒 (CF = 0.7)
  IF 白细胞升高 THEN 细菌感染 (CF = 0.8)
```

#### C-F 模型的计算规则

**① 证据的合取与析取：**
```text
合取（AND）：CF(E) = min{CF(E₁), CF(E₂), ...}
析取（OR）： CF(E) = max{CF(E₁), CF(E₂), ...}

例子：IF E₁ AND E₂ THEN H
CF(E₁) = 0.6, CF(E₂) = 0.8
CF(E) = min(0.6, 0.8) = 0.6
```

**② 规则的执行：**
```text
CF(H) = CF(H,E) × max{0, CF(E)}

如果 CF(E) < 0：规则不触发，CF(H) = 0
如果 CF(E) = 1：CF(H) = CF(H,E)（规则的可信度完全传递）

例子：IF E THEN H (0.7), CF(E) = 0.6
CF(H) = 0.7 × 0.6 = 0.42
```

**③ 多条规则的综合（同结论有多个证据）：**

当两条规则指向同一结论 H 时的合成公式：

```text
CF₁₂(H) = CF₁(H) + CF₂(H) - CF₁(H)·CF₂(H)     当两者都 > 0
CF₁₂(H) = CF₁(H) + CF₂(H) + CF₁(H)·CF₂(H)     当两者都 < 0
CF₁₂(H) = (CF₁(H) + CF₂(H)) / (1 - min(|CF₁|, |CF₂|))  当一正一负
```

**C-F 综合计算示例：**

```text
规则：
  r₁: IF E₁ THEN H (0.8)
  r₂: IF E₂ THEN H (0.9)
  r₃: IF E₁ AND E₃ THEN E₂ (0.7)

证据：
  CF(E₁) = 0.6, CF(E₃) = 0.8

计算：
  ① r₁: CF₁(H) = 0.8 × max{0, 0.6} = 0.48
  ② r₃: CF(E₂) = 0.7 × min{0.6, 0.8} = 0.42
  ③ r₂: CF₂(H) = 0.9 × max{0, 0.42} = 0.378
  ④ 综合 CF(H) = 0.48 + 0.378 - 0.48×0.378 = 0.677 ✅
```

### 知识点 14：模糊逻辑（Fuzzy Logic）🌫️

**1965 年 L.A. Zadeh 提出**，解决"非此即彼"的二元逻辑无法处理的问题。

#### 模糊集合

**核心思想**：元素可以**部分地**属于一个集合，隶属度 ∈ [0, 1]。

```text
传统集合：一个人要么"高"要么"不高"（0 或 1） ❌
模糊集合：一个人"高"的程度是 0.8 ✅

{"高"} = {0.95/180cm, 0.90/175cm, 0.85/170cm, ...}
```

**隶属函数 μ_A(x)** 表示元素 x 对集合 A 的隶属程度。

#### 模糊逻辑的基本运算

| 运算 | 公式 | 类似 |
| :---- | :---- | :---- |
| 交集（AND） | μ_A∩B(x) = min(μ_A(x), μ_B(x)) | 取小的 |
| 并集（OR） | μ_A∪B(x) = max(μ_A(x), μ_B(x)) | 取大的 |
| 补集（NOT） | μ_Ā(x) = 1 - μ_A(x) | 取反 |

#### 模糊推理示例

```text
规则：IF 温度是"高" THEN 风扇速度是"大"
输入：温度 = 35°C，对"高"的隶属度 = 0.9
输出：风扇速度调到 0.9 程度的大

这就像空调的变频控制——不是简单的"开/关"，而是"开多少" 🌡️
```

## 五、🕸️ 语义网络与知识图谱

### 知识点 15：语义网络（Semantic Network）

**1968 年提出**，用图结构表示知识。

```text
语义网络的基本元素：
  · 节点 🟢 = 概念、对象、事件
  · 边 🔗 = 关系（ISA、HAS-PART、OWNS...）

例子：
       XIAOYAN ——ISA——→ SWALLOW ——ISA——→ BIRD
         │                                    │
         └——OWNS——→ NEST-1 ——ISA——→ NEST     │
                                               │
                                         HAS-PART
                                               │
                                             WINGS
```

**语义网络的推理（继承机制）：**
```text
XIAOYAN ISA SWALLOW ISA BIRD
BIRD HAS-PART WINGS

→ 推理：XIAOYAN HAS-PART WINGS ✅
（子类自动继承父类的属性）
```

### 知识点 16：知识图谱（Knowledge Graph）—— 重点 ⭐

**知识图谱 = 语义网络的大规模升级版** 🚀

#### 核心概念

| 元素 | 含义 | 例子 |
| :---- | :---- | :---- |
| **实体（Entity）** 🧑 | 现实世界中的对象 | 特朗普、华中科技大学 |
| **概念（Concept）** 📋 | 实体的类别 | 人、大学、国家 |
| **关系（Relation）** 🔗 | 实体之间的联系 | spouse_of、located_in |
| **属性（Property）** 📊 | 实体的特征 | 年龄、身高、颜色 |

#### 著名知识图谱项目

| 项目 | 规模 | 特点 |
| :---- | :---- | :---- |

| **ConceptNet** 🌐 | 众包构建的常识知识库 | 基于 Open Mind Common Sense |
| **Freebase/Wikidata** 📚 | 49,915,906 实体 | 维基百科的结构化版本 |
| **Dbpedia** 🌍 | 127 万实体 | 从维基百科提取的结构化数据 |
| **Google KG** 🔍 | 5.7 亿实体、180 亿关系 | 2012 年推出，用于增强搜索 |
| **Probase** 🧮 | 1200 万概念 | 微软基于 Bing 的 IsA 关系提取 |

#### 知识图谱的构建流程

```text
① 实体识别   ② 关系抽取   ③ 实体链接
  ④ 知识融合   ⑤ 质量评估   ⑥ 知识更新
```

#### 知识图谱补全（Knowledge Graph Completion）

```text
知识图谱通常是不完整的，需要预测缺失的关系：

？→(妻子)→ 特朗普          → 梅拉尼娅
奥巴马 →(配偶)→ ？          → 米歇尔
中国 →(首都)→ ？            → 北京

三种预测：
· (h, r, ?) — 尾实体预测："中国的首都是什么？"
· (?, r, t) — 头实体预测："谁的首都是北京？"
· (h, ?, t) — 关系预测："中国和北京之间是什么关系？"
```

### 1. 生物启发
- **生物视觉信息处理**：
  - 腹侧通路（V1、V2、V4）：处理形状、颜色等信息
  - 背侧通路（V1、V2、V5）：处理运动等信息
- **生物神经元**：
  - 人脑约有 10¹¹ 个神经元，每个神经元与 10³~10⁴ 个其他神经元相连
  - **工作状态**：
    - 兴奋：细胞膜电位 > 动作电位阈值 → 神经冲动
    - 抑制：细胞膜电位 < 阈值
  - **学习与遗忘**：突触的传递作用可增强或减弱（可塑性）

### 2. 连接主义（Connectionism）核心思想
- 又称仿生学派 / 生理学派，源于仿生学
- 高级智能行为是从大量神经网络的连接中**自发涌现**
- 主要原理：神经网络结构 + 连接机制 + 学习算法

### 3. 连接学派发展历程（三起两落）
| 时期 | 标志事件 |
|------|---------|
| 第一次兴起（1940s-60s） | MP模型、感知器 |
| 第一次低谷（70s） | XOR问题、AI寒冬 |
| 第二次兴起（80s-90s初） | Hopfield网络、BP算法 |
| 第二次低谷（90s中-2000s） | 维度灾难、SVM崛起 |
| 第三次兴起（2006-至今） | 深度学习、AlexNet |

## 二、神经网络基本概念与近代史

### 1. 人工神经网络发展简史时间线
| 年份 | 事件 |
|------|------|
| 1943 | McCulloch-Pitts MP 模型 |
| 1958 | Rosenblatt 感知器模型 |
| 1960 | Widrow & Hoff ADALINE |
| 1969 | Minsky-Papert 《Perceptrons》——XOR问题 |
| 1982 | Hopfield 网络（HNN） |
| 1986 | Rumelhart-Hinton-Williams BP算法 |
| 1995 | Vapnik SVM |
| 2006 | Hinton《Science》深度神经网络（DBN） |
| 2012 | AlexNet 赢得 ImageNet ILSVRC |

### 2. MP 模型（McCulloch-Pitts, 1943）
- **思想**：简化生物神经元，建模人工神经元
- **特点**：
  - 多输入单输出
  - 输入分兴奋性 / 抑制性
  - 具备空间整合特性和阈值特性
  - **布尔输入和输出**（0或1）
  - **阶跃响应函数**
  - **没有可学习的权重**
- **意义**：验证型模型——证明神经元可以做计算
- **可实现**：逻辑与、逻辑或

### 3. 感知器模型（Perceptron, Rosenblatt 1958）
- **特点**：
  - **实值的输入和输出**（区别于 MP 的布尔）
  - 阶跃响应函数
  - **有可学习的权重**
- **几何解释**：通过一个（超）平面将输入空间划分为两个半空间
- **限制**：
  - 属于线性模型，**无法解决线性不可分问题**（如 XOR）
  - 即使少量不可分样本也会导致学习不收敛

### 4. ADALINE 模型（Widrow & Hoff, 1960）
- **全称**：ADAptive LInear NEuron
- **特点**：
  - 激活函数：由阶跃函数变为**连续函数**
  - 增加了 **Quantizer（量化函数）** 用于类别预测
  - 定义了**代价函数**概念（均方误差 MSE）
  - 使用**梯度下降**优化权重
- **意义**：是多层网络中 BP 算法的先驱

### 5. XOR 问题与第一次 AI 寒冬
- 1969 年 Minsky & Papert 在《Perceptrons》中指出：2 层感知器无法解决 XOR
- XOR 问题**不是线性可分**的
- **解决方法**：用三层感知器可以解决
- **AI 寒冬原因**：
  - 机器翻译进展熄火
  - 《Perceptrons》对感知器的批判
  - Lighthill report（1973）批评 AI

### 6. Hopfield 神经网络（HNN, 1982）
- AI 寒冬后的复兴
- 1987 年贝尔实验室基于 HNN 研制神经网络芯片
- **能力**：联想记忆、神经优化
- **结构**：递归 / 反馈神经网络（输出到输入有反馈连接）
- 每个吸引子（稳定状态）储存一个模式（pattern）

### 7. BP 神经网络（1986）
- 也称多层感知器（MLP）
- **历史**：BP 算法实际由 Paul Werbos 1974 年博士论文首次论证；1986 年 Rumelhart-Hinton-Williams 在 Nature 上发表使其广为人知
- **意义**：解决了多层网络的训练问题
- **万能近似定理**（George Cybenko, 1989）：多层前馈网络可近似任意函数
- **存在的问题**：
  - **维度灾难**：全连接前馈网络参数空间维数大幅增加
  - **泛化问题**：高维空间多峰非线性优化难以找到全局最优
  - 在 1990s 算力有限的情况下难以扩展

### 8. SVM 的崛起（1995, Vapnik）
- 利用**核函数**处理非线性可分问题（神经网络则用隐藏层变换）
- 优势：无需调参、速度快、全局最优解
- 标志神经网络第二次进入低谷
- **趣事**：
  - 神经网络低谷时改名"并行分布式处理"（Parallel Distributed Processing）
  - SVM 在 1995 年复兴时取名"Support Vector Network"蹭热点

### 9. 深度学习时代的开启
- **2006 年 Hinton 在 Science 上发文**：用 **DBN（深度信念网络）** 在未标注数据上预训练深度神经网络
- DBN 是首批成功应用深度架构训练的非卷积模型，MNIST 上超越 SVM
- 标志性事件：
  - 2006 手写字符识别 MNIST
  - 2010 语音识别
  - **2012 AlexNet** ImageNet ILSVRC 冠军（8 层、Top5 error 16.4%）
  - 2016 AlphaGo 击败李世石
- **成功原因**：
  - 算法哲学：模块化 + 层次化、拓扑叠加、分而治之
  - 算法细节：ReLU、Dropout、Pooling
  - 工程成本：深度学习工具大量出现，降低门槛
- **存在的问题**：
  - 暴力美学：拼数据、拼资源
  - "炼丹"：可解释性差
  - 与符号主义的融合（先验知识表达和融入）

## 三、神经网络基本思想与结构

### A. 前馈神经网络（近似函数）

#### 1. 基本概念
- 又称多层感知器（MLP）
- **目标**：通过映射 y = f(x; θ)，调整 θ 去逼近函数 f*
- **关键创新**：增加了**非线性激活单元**，使网络拥有非线性建模能力

#### 2. 网络结构
- **三种网络层**：
  - 输入层（含偏置单元）
  - 隐含层（个数和神经元数为超参数，含偏置单元）
  - 输出层
- **连接特性**：
  - 相邻层全连接（也称全连接层）
  - 同层无连接
  - 单向连接（信息可双向传播）

#### 3. 非线性激活函数
- **Logistic（Sigmoid）**：输出恒 > 0
- **Tanh**：零中心化
- **作用**：使网络拥有非线性建模能力

#### 4. 前向传播 vs 后向传播
| | 前向传播 | 后向传播 |
|---|----------|----------|
| 方向 | 输入层 → 输出层 | 输出层 → 输入层 |
| 用途 | 计算输出 | 更新参数 |
| 核心 | 加权求和 + 激活 | **链式法则** |
| 驱动 | 输入 | **损失函数**（如均方误差） |

#### 5. 通用近似定理
- 多层前馈网络可以近似任意函数
- 两种方式：
  - 增多层数 / 减少隐层神经元
  - 只需三层 / 隐层神经元个数多

#### 6. 前馈神经网络特点
- **优点**：
  - 强大逼近特性、通用函数拟合
  - 较好的容错性
- **缺点**：
  - 收敛速度慢
  - 可能陷入局部极值
  - 没有选择隐层数和节点数的通用准则

---

### B. Hopfield 网络（解决优化问题）

#### 1. HNN 概述
- 单层、全连接、固定参数的反馈型神经网络
- 输出到输入均有反馈连接
- 每个神经元既是输入也是输出
- 引入**能量函数**判断网络稳定性
- 分两种：
  - **离散型（DHNN）**：联想记忆
  - **连续型（CHNN）**：优化问题

#### 2. 离散 Hopfield 网络（DHNN）
- 二值神经元：{0,1} 或 {-1,1}（抑制/激活）
- 各节点**没有自反馈**
- **两种工作方式**：
  - 异步（串行）：t 时刻只有一个神经元状态变化
  - 同步（并行）：所有神经元同时变化
- **能量函数**：
  $$E = -\frac{1}{2}\sum_{i=1}^{N}\sum_{j=1,j\neq i}^{N}w_{ij}v_iv_j + \sum_{i=1}^{N}\theta_i v_i$$
- 系统运行中能量不断减少，最终达到极小值（稳定态）
- W 是对角元为 0 的对称矩阵 → 串行稳定性
- **应用**：联想记忆（但存在**串扰**问题）

#### 3. 连续 Hopfield 网络（CHNN）
- 拓扑结构与 DHNN 一致
- **更接近生物神经元**
- 神经元状态取 0~1 连续值
- 激活函数：**S 型连续函数**（非阶跃）
- 神经元同步工作
- 稳定条件：传递函数连续有界 + 权重矩阵对称
- **应用**：1985 年 Hopfield & Tank 求解 TSP 问题
  - 思路：将代价函数写为 Hopfield 能量函数形式 → 平衡态即为问题的解

#### 4. 玻尔兹曼机（Boltzmann Machine）
- 解决 Hopfield 串扰和局部最优问题
- 通过**概率分布**让单元状态变化，避免陷入局部最优
- 保留 Hopfield 假设：权重对称、自身无连接、二值输出
- **新增**：可见单元 + 隐藏单元（隐变量带来更强表达能力）

#### 5. 受限玻尔兹曼机（RBM）
- 由可见层和隐藏层构成
- **层内无连接，层间全连接**
- 信息双向流动
- **学习目标**：最大化似然
- 解决了含隐变量的 BM 训练困难问题

#### 6. 深度玻尔兹曼机（DBM）
- 多个 RBM 堆叠 → 隐藏层变多
- **训练方式**：贪婪、逐层、无监督预训练
- 适用于**无监督学习**（无向连接）

#### 7. 深度信念网络（DBN）
- 靠近可视层用贝叶斯信念网络（**有向**），最远端用 RBM（**无向**）
- **是一种生成模型**，深层概率有向图模型
- **训练**：逐层无监督预训练 + 标注数据精调
- 适用于**监督学习**
- **意义**：2006 年 Hinton 在 Science 上证明深度架构有效（MNIST 上超 SVM）
- **应用**：识别特征、分类数据、生成数据（如生成手写数字）

---

### C. 循环神经网络 RNN（近似动力学系统）

#### 1. 序列化数据
- 时序信号特点：上下文强关联、序列长度变化
- CNN 难以直接建模
- 典型：音频、视频、文本、生物信号

#### 2. 序列建模需求
- 处理变长序列
- 保留序列顺序
- 建模上下文依赖
- 整个序列共享模型参数

#### 3. RNN 简史
- 1986 Jordan Network
- 1990 Elman Network
- 1994 RNN 概念首次提出
- 1997 LSTM（Schmidhuber）
- 2010 RNN 用于语言模型
- 2009-2012 Schmidhuber 团队多次获得国际竞赛冠军

#### 4. RNN 基本结构
- 三种网络层：输入层、隐含层、输出层
- 两种表示：折叠表示、展开表示
- 隐藏状态：
  $$s_t = f(s_{t-1}, x_t)$$
- 特征 sₜ 不仅取决于当前输入 xₜ，还取决于 sₜ₋₁

#### 5. RNN 与非线性动力系统
- RNN 可理解为通用的非线性系统
- **通用近似定理**：完全连接的 RNN 可近似任何非线性动力系统

#### 6. RNN 典型应用
| 应用 | 类型 |
|------|------|
| 情感分类 | 序列 → 类别 |
| 中文分词 | 同步序列 → 序列（B/M/E/S 标注） |
| 信息抽取 | 同步序列 → 序列（B/I/O 标注） |
| 机器翻译 | 异步序列 → 序列 |
| 图像描述（Image Caption） | 图像 → 序列 |
| 手写体识别 | 序列 → 序列 |
| 计算机合成音乐 | 生成 |

#### 7. RNN 特点
- **优点**：
  - 处理任意长度序列
  - **参数共享**（不同时间步用相同权重）
  - 通用计算能力（按任意精度逼近非线性系统）
- **缺点**：
  - **建模单向性**：输出仅与 t 时刻之前输入有关
  - **长序列记忆丢失（梯度消失）**

#### 8. 梯度消失/爆炸的原因
- 反向传播时，根据链式法则，激活函数求导值需要**连乘**
- 不同时刻参数共享，导致重复对相同矩阵 W 求 t 次幂
- 通过特征值分解 W = V·diag(λ)·V⁻¹：
  - W^t = V·diag(λ^t)·V⁻¹
  - λ > 1 → 梯度爆炸（少见）
  - λ < 1 → 梯度消失（**大部分情况**）
- BP 算法推广为 BPTT（Back Propagation Through Time）

#### 9. 改善梯度消失的途径
- 改进 RNN 结构（**LSTM、GRU**）
- 使用合适的激活函数（如 tanh）

---

### D. 长短期记忆网络（LSTM, 1997）

#### 1. 核心贡献
- 引入**自循环**，产生梯度长时间持续流动的路径（gradient path）
- 解决梯度消失的根本原因

#### 2. 状态设计
- **细胞状态 C**：长记忆
- **隐藏状态 h**：短记忆

#### 3. 三个门控机制（gate）
| 门 | 作用 |
|----|------|
| **遗忘门**（forget gate, fₜ） | 控制自循环权重，对上一节点输入选择性忘记 |
| **输入门**（input gate, iₜ） | 选择性记忆当前阶段输入 |
| **输出门**（output gate, oₜ） | 控制 LSTM 单元的输出 |

#### 4. 输入输出
- 输入：cₜ₋₁、hₜ₋₁、xₜ
- 输出：cₜ、hₜ、yₜ

#### 5. 计算流程
1. **遗忘门**：fₜ = σ(W_f·[hₜ₋₁, xₜ] + b_f)
2. **输入门**：
   - iₜ = σ(W_i·[hₜ₋₁, xₜ] + b_i)
   - C̃ₜ = tanh(W_C·[hₜ₋₁, xₜ] + b_C)
3. **细胞状态更新**：cₜ = fₜ ⊗ cₜ₋₁ + iₜ ⊗ C̃ₜ
   - 有选择地忘记 + 有选择地记忆
4. **输出门**：
   - oₜ = σ(W_o·[hₜ₋₁, xₜ] + b_o)
   - hₜ = oₜ ⊗ tanh(cₜ)

#### 6. LSTM 与 RNN 对比
| | 联系 | 区别 |
|---|------|------|
| LSTM vs RNN | 都是循环链式结构；都假设单向依赖 | LSTM 引入大量非线性单元；增加细胞状态 C 和门控；选择性建模长期记忆 |

---

### E. 双向 RNN
- 时刻 t 输出**同时取决于过去和未来**信息
- 每个隐藏层有两个节点：
  - 一个正向计算
  - 一个反向计算
- 输出层由两值决定
- **正向和反向计算的权重不共享**

---

## 四、关键考点小结

### 1. 各神经元/网络模型对比
| 模型 | 输入输出 | 激活 | 权重学习 | 主要贡献 |
|------|---------|------|----------|---------|
| MP (1943) | 布尔 | 阶跃 | 否 | 验证模型可计算 |
| Perceptron (1958) | 实值 | 阶跃 | 是 | 引入可学权重 |
| ADALINE (1960) | 实值 | 连续 | 梯度下降 | MSE代价函数、梯度下降 |
| BP-MLP (1986) | 实值 | 非线性（Sigmoid/Tanh） | BP算法 | 解决多层训练 |
| Hopfield | 二值/连续 | 阶跃/S型 | 固定 | 联想记忆/优化 |
| RBM | 二值 | - | 最大似然 | 无监督预训练 |
| RNN | 序列 | tanh | BPTT | 序列建模 |
| LSTM | 序列 | tanh+门控 | BPTT | 解决梯度消失 |

### 2. 三次兴起 & 两次低谷的根本原因
- 第一次低谷：**线性不可分问题**（XOR）
- 第二次低谷：**算力和模型不足 + SVM 竞争**
- 第三次兴起：**深度学习 + 大数据 + GPU 算力**

### 3. 重要公式
- **Hopfield 能量函数**：E = -½ΣΣwᵢⱼvᵢvⱼ + Σθᵢvᵢ
- **LSTM 细胞更新**：cₜ = fₜ⊗cₜ₋₁ + iₜ⊗C̃ₜ
- **RNN 隐状态**：sₜ = f(sₜ₋₁, xₜ)

### 4. 三个基本架构对比
| 架构 | 解决问题 | 拓扑结构 |
|------|---------|---------|
| 前馈网络 | 函数近似 | 单向、分层 |
| Hopfield | 优化问题、联想记忆 | 全连接反馈 |
| RNN | 动力学系统 / 序列建模 | 循环（带时序） |

```text
传统方法（SIFT/HOG） → 深度学习崛起（AlexNet/ImageNet）
         ↓
    🏗️ CNN 三大核心组件
    ┌─────────────┬──────────────┬──────────────┐
    │  卷积层      │   池化层      │   全连接层     │

    📈 经典架构演进（LeNet → AlexNet → VGG → GoogLeNet → ResNet → ...）
```

## 一、📜 CNN 是怎么来的？

### 知识点 1：传统方法怎么识别图片？（了解背景）

在 CNN 火之前，人们靠 **手工设计特征** 来识别图片：

| 方法 | 一句话解释 |
| :---- | :----------- |
| **SIFT** 🧩 | 找图片中的"关键点"（角、边、纹理斑块），不管图片放大缩小都能认出来 |
| **HOG** 📊 | 统计图片局部区域里"梯度方向"的分布，看轮廓走向 |
| **Bag-of-Words** 📚 | 把图片拆成"视觉单词"，像查词典一样数每种"单词"出现了多少次 |

> ❌ **痛点**：这些特征都是人手动设计的，换个场景就不好使了，泛化能力差。

### 知识点 2：深度学习为什么突然火了？（关键转折）

**2012年，AlexNet 在 ImageNet 比赛上一战封神 🏆**

| 对比 | 传统方法 | AlexNet（深度学习） |
| :---- | :------- | :----------------- |
| Top-5 错误率 | 26.2% | **15.3%**（碾压！） |
| 特征怎么来的 | 人手工设计 | **让网络自己学** 🤯 |

**成功三要素（缺一不可）：**

1. **🔥 大数据** — ImageNet 数据集：120 万张图片，1000 个类别（李飞飞团队用亚马逊众包标注的）
2. **🔥 大算力** — GPU 并行计算，训练速度是 CPU 的 50 倍
3. **🔥 好算法** — ReLU 激活函数、Dropout 防过拟合、数据增强

> 💡 **为什么 LeNet（1998）当时没火？** — 因为没赶上大数据+大算力的时代，生不逢时 😅

## 二、🧬 CNN 的灵感从哪来？

### 知识点 3：生物视觉系统的启发

**Hubel & Wiesel（1958, 1981 诺贝尔奖）** 研究猫的视觉皮层，发现：

- 🐱 **视觉皮层神经元只对视野中一小块区域有反应** → 这就是 **局部感受野**
- 🐱 **简单细胞**检测边缘/方向，**复杂细胞**检测运动/位置不变性
- 🐱 **视觉信息逐层处理**：视网膜 → V1（初级）→ V2 → V4 → 前额叶

> 💡 **CNN 的设计就是模仿这套视觉系统**：底层看边缘 → 中层看形状 → 高层看整个物体

### 知识点 4：CNN 发展时间线（谁在什么时候做了什么）

```text
1980  🧪 Fukushima → Neocognitron（CNN 的雏形诞生）
1998  🧪 LeCun → LeNet-5（手写数字识别，第一次实用化）
2012  💥 Krizhevsky + Hinton → AlexNet（深度学习大爆发！）
2014  📈 Simonyan → VGGNet（更深了：16~19层）
2014  📈 Szegedy → GoogLeNet（更宽了：多尺度卷积并行）
2015  🚀 何恺明 → ResNet（革命：152层也能训练了！）
2017  🚀 DenseNet（密集连接，特征充分利用）
2018  🚀 SeNet（通道注意力机制，学会"关注重点"）
```

## 三、🏗️ CNN 的三个核心组件（重中之重 ⭐）

### 知识点 5：卷积层 🔎 —— "特征提取器"

**用一个小窗口滑过整张图，提取局部特征。**

就像你用放大镜 📲 一个字一个字地扫描报纸：

```
输入图片（5×5）    卷积核（3×3）        输出特征图（3×3）
  🟦🟦🟦🟦🟦        🟥🟥🟥
  🟦🟦🟦🟦🟦        🟥🟥🟥     ═══►    算出新数值
  🟦🟦🟦🟦🟦        🟥🟥🟥
  🟦🟦🟦🟦🟦                             每个位置都算一遍
  🟦🟦🟦🟦🟦
```

**三个关键参数（考试必考 🎯）：**

| 参数 | 含义 | 举个栗子 🌰 |
|------|------|------------|
| **Kernel Size** 🪟 | 窗口有多大 | 3×3、5×5、11×11 |
| **Stride** 🚶 | 一次滑几步 | stride=1 慢慢滑，stride=2 跳跃 |
| **Padding** 📦 | 图片边缘要不要补0 | valid（不补，图变小）/ same（补0，图不变） |

**📐 计算输出尺寸的公式（必背！）：**

```
输出尺寸 = (输入尺寸 - 卷积核大小 + 2 × Padding) ÷ Stride + 1
```

> 举个栗子：输入 224×224，卷积核 5×5，stride=1，padding=0
> → 输出 = (224 - 5 + 0) ÷ 1 + 1 = **220×220**

**一个卷积层到底干了啥？**
```
输入: H × W × N（高×宽×通道数）
卷积核: K × K × N（每个核看所有通道），一共 M 个这样的核
输出: R × C × M（M 个特征图，每个图是 R×C）
```

---

### 知识点 6：池化层 📉 —— "压缩器"

**把图片"缩小"，保留最重要的信息，扔掉不重要的细节。**

就像把一张高清照片压缩成缩略图 🖼️ → 📌

```
最大池化（Max Pooling）— 取最大值：
  2×2 窗口，步长=2：
  [1  5]      [8  6]        最大池化后：
  [3  8]      [2  4]    →   [8  6]
  [6  2]      [7  9]         [7  9]
  [3  1]      [5  0]
保留每个区域的"最强者"
```

**为什么需要池化？**
- ✅ **减少参数量** — 图片尺寸变小了，后面的计算量就少了
- ✅ **增强平移不变性** — 物体稍微动一下，池化后的结果差不多

---

### 知识点 7：全连接层 🧠 —— "分类决策器"

**把前面提取好的特征综合起来，做出最终判断。**

```
卷积层 + 池化层                   全连接层
（提取特征）                       （做判断）
  🔎📉                              🧠
  [特征图]  →  拉直成一串数字  →  [FC] → [FC] → "这是一只猫 🐱"
```

> 💡 简单理解：卷积层和池化层负责"看"，全连接层负责"想"。

---

### 知识点 8：激活函数 ⚡ —— "让网络学会非线性"

CNN 里只用卷积和全连接的话，无论叠多少层都只是"线性变换"的堆叠，表达能力有限。**激活函数引入非线性，让网络能学到复杂模式。**

| 函数 | 公式 | 特点 |
|------|------|------|
| **ReLU** 👍 | f(x) = max(0, x) | 负数直接变0，正数不变；计算快、缓解梯度消失 |
| **Sigmoid** 🐢 | f(x) = 1/(1+e⁻ˣ) | 老派方法，深网络中容易梯度消失 |
| **Tanh** 🐢 | f(x) = (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ) | 输出范围[-1,1]，也有梯度消失问题 |

> ⭐ **ReLU 是 AlexNet 成功的关键因素之一**，因为它解决了深层网络的梯度消失问题。

---

### 知识点 9：归一化层 📏 —— "让训练更稳定"

**Batch Normalization（BN）**：每次训练时，把数据拉回到均值为0、方差为1的分布。

```
🛠️ 有什么用？
  ✅ 训练更快（可以用更大的学习率）
  ✅ 缓解梯度消失
  ✅ 自带一点正则化效果（防过拟合）
  
📌 怎么理解？
  就像给每个中间层的输出"做一次标准化"，不让数值过大或过小
```

> 💡 **Internal Covariate Shift** 问题：网络在训练时，每层的输入分布一直在变化，就像"移动靶"，很难瞄准。BN 就是把这个"移动靶"固定住 🎯。

## 四、💪 CNN 的两大制胜法宝（必考题 ⭐）

### 知识点 10：局部连接 vs 参数共享

**问题：为什么 CNN 比全连接网络好？🤔**

假设输入是一张 **1000×1000 的图**，隐层有 **100 万个神经元**：

| 方式 | 参数量 | 对比 |
|------|--------|------|
| **全连接** ❌ | 1000×1000×1,000,000 = **10¹²**（太大了！） | 💸 根本算不动 |
| **局部连接** ✅ | 每个神经元只看 10×10 区域 → **10⁸** | 📉 少了 1 万倍 |
| **局部连接 + 参数共享** ✅✅ | 整个图用一个卷积核 → **10⁴**（1 万） | 🚀 再少 1 万倍 |

**核心思想：**
- 🪟 **局部连接**：每个神经元只看图片的一小块（感受野），模仿生物视觉
- 🔄 **参数共享**：同一个卷积核（特征检测器）滑遍整张图，不管物体在图的哪个位置，都用同一套参数来检测

> 🤯 从 **10¹² 到 10⁴**，参数减少了 **1 亿倍**！这就是 CNN 能成功的关键。

### 知识点 11：层级特征学习

CNN 不是一下子就认出物体的，而是**逐层抽象**：

```
底层（浅层） 🧱        中层 🧩             高层 🎯
  ┌──────┐         ┌──────┐            ┌──────┐
  │ 边缘  │   →    │ 形状  │    →       │ 物体  │
  │ 颜色  │         │ 部件  │            │ 场景  │
  │ 纹理  │         │ 图案  │            │ 类别  │
  └──────┘         └──────┘            └──────┘
  
  比如识别人脸 👤：
  第1层：看到边缘、点、色块
  第2层：看到眼睛、鼻子、嘴巴
  第3层：认出这是一张"脸"
```

## 五、📊 经典 CNN 架构演进（逐个击破）

### 知识点 12：LeNet-5（1998）—— 🥚 CNN 的"始祖"

| 项目 | 内容 |
|------|------|
| **谁做的** | Yann LeCun（深度学习三巨头之一） |
| **用来干嘛** | 识别手写数字（如邮局信封上的邮编 📮） |
| **特点** | 6 万参数，Sigmoid 激活，平均池化 |
| **结构** | 卷积 → 池化 → 卷积 → 池化 → 全连接 |

> 💡 **LeNet 在 1998 年就做出来了，但当时没火**，因为数据不够多、算力不够强。实力也需要时代舞台！

### 知识点 13：AlexNet（2012）—— 💥 深度学习的"原子弹"

**历史意义**：在 ImageNet 比赛上把第二名（26.2%）远远甩开（15.3%），让全世界看到了深度学习的威力。

**比起 LeNet 改进在哪？**

| 改进点 | 效果 |
|--------|------|
| **更深（8 层）** | 学到更复杂的特征 |
| **ReLU 激活函数** ✅ | 解决梯度消失，训练更快 |
| **Dropout** 🎲 | 随机"丢弃"一些神经元，防过拟合 |
| **GPU 并行训练** 🎮 | 训练速度是 CPU 的 50 倍 |
| **数据增强** 🔄 | 把图片翻转、旋转、裁剪，变出更多训练数据 |

**AlexNet 结构速览**（8 层，6200 万参数）：

```
输入 227×227×3
  ↓
CONV1（11×11, stride=4）→ 55×55×96   约 3.5 万参数
  ↓  Pool1（3×3）
CONV2（5×5）→ 27×27×256              约 61 万参数
  ↓  Pool2（3×3）
CONV3（3×3）→ 13×13×384              约 89 万参数
CONV4（3×3）→ 13×13×384              约 133 万参数
CONV5（3×3）→ 13×13×256              约 88 万参数
  ↓  Pool3（3×3）
FC1 → 4096 神经元                     约 3775 万参数 💰
FC2 → 4096 神经元                     约 1678 万参数 💰
FC3 → 1000 类（输出分类结果）
```

> ⚡ **有趣的事实**：全连接层（FC1+FC2）占了总参数量的 **87%**，但卷积层占了大部分**计算量**。

### 知识点 14：VGGNet（2014）—— 📏 "更深、更规整"

| 项目 | 内容 |
|------|------|
| **核心思想** | **全用 3×3 小卷积核**，堆更深（16/19层） |
| **为什么用小的？** | 2个3×3 ≈ 1个5×5的感受野，但参数更少、非线性更强 |

> 🧮 **感受野计算**：3 个 3×3 卷积层堆叠 → 感受野 7×7，但参数量（27）比直接用 1 个 7×7（49）少 45%。

### 知识点 15：GoogLeNet / Inception（2014）—— 🎪 "在同一层用多种尺度的眼睛"

| 项目 | 内容 |
|------|------|
| **核心思想** | 同一层并行使用 1×1、3×3、5×5 卷积，让网络自己选 |
| **1×1 卷积的作用** | 降维！减少计算量 🔽 |

> 💡 **类比**：看一张图时，有人凑近看细节（小卷积核），有人站远看整体（大卷积核），把大家的意见综合起来。

### 知识点 16：ResNet（2015, 何恺明）—— 🚀 "让网络可以无限深"

**核心问题**：网络并不是越深越好。实验发现，56 层的网络比 20 层的**训练误差还大**，这叫 **退化问题（Degradation Problem）**。

**ResNet 的解决方案：残差学习 + 跳跃连接**

```
传统网络学的是：        H(x) = 函数
ResNet 学的是：         F(x) = H(x) - x  （残差）
最终输出：              H(x) = F(x) + x  （加上输入本身）

图示：
  输入 x
    │
    ├──────────────────┐
    │                   │
  [卷积层]             │（跳跃连接，短路直达）
    │                   │
  [卷积层]             │
    │                   │
    └────→ ➕ ←────────┘
            │
          输出 H(x) = F(x) + x
```

**为什么这招好用？**
- 🔗 如果深层学不到新东西，可以让 F(x) → 0，直接"抄作业"输出 x
- 🏃 梯度可以通过跳跃连接直接回传到浅层，解决梯度消失
- ✅ 152 层甚至上千层都能正常训练

> 💡 **一个形象的比喻**：传统网络像叠积木，越高越不稳。ResNet 相当于给每块积木加了一根"安全绳"，就算上面没搭好，下面也能兜住。

### 知识点 17：后续发展（了解即可）

| 模型 | 年份 | 一句话说清 |
|------|------|-----------|
| **DenseNet** 🌐 | 2017 | 每层和前面**所有层都连接**，特征都用上 |
| **SeNet** 👀 | 2018 | 学会"注意力"，知道哪些通道重要，哪些不重要 |
| **Capsule Network** 💊 | 2017+ | 解决 CNN 不理解"空间关系"的问题（如：眼睛在嘴巴上面还是下面？） |

## 六、🎯 重要公式与计算速查

### 📐 卷积输出尺寸公式（必背！）

```
输出尺寸 = (W - K + 2P) / S + 1

W = 输入尺寸     K = 卷积核大小
S = 步长         P = 填充数
```

### 🧮 计算举例（必练！）

**例题**：输入 224×224×3 的彩色图像（RGB 3 通道）

> 第一步：用 32 个 5×5×3 的卷积核，stride=1，padding=0
> - 输出 = (224 - 5 + 0) / 1 + 1 = **220×220×32**
>
> 第二步：用 64 个 5×5×32 的卷积核
> - 输出 = (220 - 5 + 0) / 1 + 1 = **216×216×64**

### 感受野对比

| 方式 | 参数量 | 感受野 | 谁更好？ |
|------|--------|--------|---------|
| 直接用 7×7 | 49 | 7×7 | 参数多 ❌ |
| 堆叠 3 个 3×3 | 27 | 7×7 | 省参数、非线性多 ✅ |

### AlexNet 的参数量 vs 计算量

| 类型 | 主要来源 | 数值 |
|------|---------|------|
| **参数（Weights）** 💾 | 全连接层（FC1+FC2） | 约 **6200 万** |
| **计算量（MACs）** ⚡ | 卷积层 | 约 **1.1G** |

> 💡 **记忆口诀**：**卷积层费计算、全连接层费参数**

## 七、💡 实践中的要点

### 知识点 19：数据和算力是关键

- 📊 **数据质量 > 模型复杂度** — 再好的模型，没有好数据也白搭
- 🖥️ **训练成本极高** — ChatGPT-4 用了 25000 张 A100 显卡训练了 3 个多月

### 知识点 20：常用优化技巧

| 技巧 | 一句话解释 |
|------|-----------|
| **数据增强** 🔄 | 把图片翻转、旋转、裁剪、变色，变出更多训练数据 |
| **迁移学习** 🎁 | 拿别人已经训好的模型，在自己的小数据集上"微调" |
| **学习率衰减** 📉 | 一开始步子迈大点，后期步子小点，更精准 |
| **Dropout** 🎲 | 训练时随机"关掉"一些神经元，强迫网络学会冗余特征 |

## 第一部分：🎨 GAN（生成对抗网络）

### 知识点 1：GAN 的核心思想

**GAN = 生成器（Generator） + 判别器（Discriminator）**，两者互相对抗、共同进步 🤼

| 角色 | 名字 | 干什么的 | 比喻 🎭 |
|------|------|---------|---------|
| **生成器 G** 🖌️ | 伪造者 | 从随机噪声 z 生成假图片，骗过判别器 | 假钞制造犯 |
| **判别器 D** 🔍 | 鉴别师 | 判断输入是"真"还是"假"（来自数据集还是 G 生成的） | 验钞机 |

**训练过程就像警察抓小偷 👮‍♂️ 和小偷升级手艺 🦹：**
- 小偷（G）努力造更逼真的假币
- 警察（D）努力分辨真币假币
- 互相促进，最后 G 能造出以假乱真的图片

> 💡 **2014 年 Ian Goodfellow 提出**。据说 Goodfellow 在酒吧里跟朋友争论时突然想到这个想法，当晚就写代码验证成功了 🍺→💡

### 知识点 2：GAN 的训练目标（Minimax Game）

**目标函数：**
```
min_G max_D V(D,G) = E[log D(x)] + E[log(1 - D(G(z)))]
```

- D 想让 D(x) → 1（真图片判真），D(G(z)) → 0（假图片判假）
- G 想让 D(G(z)) → 1（骗过判别器）

**训练步骤（交替训练）：**
1. 固定 G，训练 D（让 D 更会分辨真假）
2. 固定 D，训练 G（让 G 更会骗 D）
3. 循环 k 步 D + 1 步 G

### 知识点 3：GAN 的经典变体 🏗️

| 模型 | 年份 | 一句话说清 | 核心创新 |
|------|------|-----------|---------|
| **DCGAN** 🎨 | 2016 | 把 CNN 引入 GAN | 用卷积代替全连接，BN、ReLU、LeakyReLU |
| **WGAN** 🌊 | 2017 | 用 Wasserstein 距离代替 JS 散度 | 解决训练不稳定、mode collapse |
| **CGAN** 🏷️ | 2014 | 给 GAN 加条件，指定生成内容 | 输入时拼接 label/条件信息 |
| **Pix2Pix** 🖼️↔️🖼️ | 2017 | 图像到图像的翻译（有配对数据） | 条件 GAN + U-Net 生成器 |
| **CycleGAN** 🔄 | 2017 | 无配对数据的图像翻译 | 循环一致性损失（cycle consistency） |
| **PGGAN** 📈 | 2018 | 渐进式训练，从低分辨率到高分辨率 | Progressive Growing |
| **StyleGAN** 💅 | 2019 | 控制生成图像的"风格" | AdaIN、Mapping Network、Mixing Regularization |
| **StyleGAN2** ✨ | 2020 | 修复 StyleGAN 的伪影 | 改进 AdaIN 实现 |

### 知识点 4：各 GAN 变体的关键理解

**DCGAN** — 第一个把 CNN 成功用于 GAN 的工作：
- 去掉全连接层，用卷积代替
- 生成器用转置卷积上采样
- Batch Normalization + ReLU（生成器）/ LeakyReLU（判别器）

**WGAN** — 解决 GAN 训练不稳定的问题：
- ❌ 不用 Sigmoid，去掉 log
- ✅ 用 Wasserstein 距离（Earth Mover's Distance）
- ✅ 权重裁剪（weight clipping）保证 Lipschitz 约束
- ✅ 优化器用 RMSProp/SGD，不用 Adam

**CycleGAN** — 没有配对数据也能做图像翻译：
- 关键 idea：**循环一致性** — 把一张图翻译成目标风格，再翻译回来，应该和原图差不多
- 比如：照片 ↔ 莫奈画作，马 ↔ 斑马 🦓

**StyleGAN** — 控制图像的"风格"：
- **Mapping Network**：把随机噪声 z 映射到中间 latent space w
- **AdaIN**（Adaptive Instance Normalization）：把风格信息注入生成过程
- **Mixing Regularization**：训练时用两种不同的风格向量混合

> 📌 **StyleGAN 的应用**：https://www.artbreeder.com（在线生成人脸）

### 知识点 5：GAN 的应用场景

- 🎨 图像生成（人脸、风景、艺术创作）
- 🖼️ 图像超分辨率、上色、修复
- 🔄 风格迁移（照片 → 动漫风格）
- 🎬 视频生成、数据增强
- 💊 药物分子设计

## 第二部分：👀 Attention（注意力机制）

### 知识点 6：注意力机制的灵感来源

**认知注意力（Cognitive Attention）**：人脑处理信息时，不会平等对待所有信息，而是**聚焦在重要的部分**。

现实中的注意力 👁️：
- 你在人群中找朋友 → 自动忽略其他人，只扫视面孔
- 读文章时 → 重点看关键词、标题

**注意力机制的核心思想：让网络学会"该看哪里"** 🎯

### 知识点 7：注意力类型

| 分类 | 说明 |
|:---- |:---- |
| **Hard Attention** 🔲 | 要么看（1）要么不看（0），不可微，难训练 |
| **Soft Attention** 🌫️ | 给每个位置一个 [0,1] 的权重，可微，端到端训练 |
| **Self-attention** 🔄 | 在自己的序列内部算注意力，"每个词看其他词" |
| **Cross-attention** 🔀 | 在两个不同序列之间算注意力（如翻译任务） |

**Bottom-up 注意力**（数据驱动的，自下而上） vs **Top-down 注意力**（任务驱动的，自上而下）

### 知识点 8：Attention 发展里程碑

```
2014  🐣 Google DeepMind → Recurrent Attention Model（图像上的注意力）
2014  🐣 Bahdanau + Bengio → Attention 用于机器翻译（NLP 开始用）
2015  🐣 Xu et al. → Show, Attend and Tell（图像描述）
2017  💥 Google → "Attention is All You Need"（Transformer 诞生，革命！）
2017  📈 SENet → 通道注意力（CV 领域）
2018  📈 CBAM → 空间 + 通道注意力
2020  💥 ViT → Vision Transformer（Transformer 用于图像分类）
```

### 知识点 9：Self-Attention（自注意力）—— 重点中的重点 ⭐

**这是 Transformer 的核心，也是当前 LLM（如 ChatGPT）的基础！**

#### 9.1 直观理解

一句话说清 Self-Attention：**序列中的每个元素，去看序列中所有其他元素，然后根据"谁更重要"来整合信息。**

类比：你在一个会议室里做决策 🏢
- **Query（Q）** 🙋 = 你问的问题（你想知道什么）
- **Key（K）** 🏷️ = 每个人的"标签"（谁有什么信息）
- **Value（V）** 📄 = 每个人实际说的内容

> **bilibili 例子**：你想在 B 站搜视频 — 你的搜索词是 Query，视频的标签是 Key，视频内容本身是 Value

#### 9.2 Self-Attention 的计算步骤

```
输入序列：x₁, x₂, x₃, x₄

Step 1：生成 Q、K、V
  Q = W_Q × X
  K = W_K × X  
  V = W_V × X

Step 2：算注意力分数（每个词和其他所有词的匹配度）
  score_ij = Q_i · K_j / √d_k    （点积 + 缩放）
  
Step 3：Softmax 归一化
  α_ij = softmax(score_ij)        （变成概率，和为 1）

Step 4：加权求和
  output_i = Σ_j α_ij × V_j      （按重要程度汇总信息）
```

#### 9.3 矩阵形式（可以看到 GPU 并行计算的优势 🚀）

```
Q = X × W_Q
K = X × W_K
V = X × W_V

Attention(Q, K, V) = softmax(Q × K^T / √d_k) × V
```

> **全部可以写成矩阵乘法，GPU 并行计算，超快！**

#### 9.4 Scaled Dot-Product Attention 为什么要除 √d_k？

当维度 d_k 很大时，点积的结果会很大，Softmax 的梯度会变得非常小（梯度消失）。除以 √d_k 相当于**把数值拉回合适的范围**。

### 知识点 10：Multi-Head Self-Attention（多头自注意力）

**一个头不够？那就多来几个！** 👥

```
不学一个注意力，而是学 h 个不同的注意力（不同头关注不同方面）：
  head_i = Attention(Q × W_Q_i, K × W_K_i, V × W_V_i)

把所有头的结果拼起来：
  MultiHead(Q, K, V) = Concat(head₁, ..., head_h) × W_O
```

> 💡 有的头关注句法关系（主谓宾），有的头关注语义关系，各司其职。

### 知识点 11：Attention 在 CV 中的应用

| 模型 | 一句话说清 |
|:---- |:----------|
| **SENet** 🧪 | Squeeze（压缩空间）-Excitation（学习通道权重）-Scale（加权），关注**哪些通道**重要 |
| **CBAM** 🔍 | 既是"where"（空间注意力 SAM）又是"what"（通道注意力 CAM），双管齐下 |
| **Non-local NN** 🌐 | CV 中的 self-attention，一个像素关注所有像素，捕获**全局**信息 |
| **ViT** 🖼️ | 把图像切成 16×16 的 patch，当成 token 序列喂给 Transformer |
| **DETR** 🎯 | Facebook 用 Transformer 做目标检测，不要锚框、不要 NMS |
| **SETR** 🏘️ | Transformer 做语义分割 |
| **PCT** ☁️ | Transformer 做点云处理 |

## 第三部分：🕸️ GNN（图神经网络）

### 知识点 12：图的基本概念

**图（Graph）** 由两部分组成：
- **节点（Vertex/Node）** 🟢 — 图中的点
- **边（Edge）** 🔗 — 点之间的连接

**图的分类：**
| 类型 | 说明 | 例子 |
|:---- |:---- |:---- |
| **有向图** ➡️ | 边有方向 (A→C ≠ C→A) | Twitter 关注（你关注他 ≠ 他关注你） |
| **无向图** ↔️ | 边没有方向 (A-B = B-A) | 微信好友（你好友他 = 他好友你） |

**为什么传统方法处理图很难？**
- CNN 需要**规则网格**（矩阵形式的图片），而图是不规则的（Non-Euclidean Structure）🚫
- 每个节点邻居数量不一样，没法用固定大小的卷积核

### 知识点 13：GNN 的核心思想

**核心 idea：每个节点通过不断聚合邻居的信息来更新自己的表示** 🔄

```
节点 v 的隐藏状态 h_v 由以下信息决定：
  ① 自己的特征 x_v
  ② 相邻边的特征 x_(u,v)  
  ③ 邻居节点的状态 h_u

公式：h_v = f(x_v, x_(u,v), h_u, x_u)
```

**GNN 和 RNN 的关系**：GNN 在图上迭代传播信息，类似于 RNN 在时间序列上展开，早期 GNN 确实用 RNN 来实现。

### 知识点 14：GCN（图卷积网络）—— GNN 的里程碑

**GCN = Graph Convolutional Networks**：把 CNN 的卷积操作"搬到"图结构上 🚚

**CNN vs GCN：**
| | CNN | GCN |
|:--|:----|:----|
| 数据结构 | 规则网格（图片） | 不规则图结构 |
| 邻居 | 固定数量的相邻像素 | 数量不等的相邻节点 |
| 卷积方式 | 固定卷积核滑动 | 聚合邻居信息 + 归一化 |

**GCN 的核心公式（简化版）：**
```
H^(l+1) = σ(Ã · H^(l) · W^(l))

Ã = 归一化的邻接矩阵
H^(l) = 第 l 层的节点特征
W^(l) = 第 l 层的权重（可学习参数）
σ = 激活函数（如 ReLU）
```

### 知识点 15：图上的常见任务

| 任务 | 例子 |
|:---- |:---- |
| **节点分类** 🏷️ | 预测用户的兴趣标签 |
| **图分类** 🖼️ | 判断分子是否有毒 |
| **链接预测** 🔗 | 预测两个用户会不会成为好友 |
| **边分类** 📊 | 判断两个节点之间的关系类型 |

### 知识点 16：GNN 的实际应用 🌍

| 领域 | 应用 |
|:---- |:---- |
| 🖼️ **场景图生成**（Scene Graph） | 图片 → "人在骑🐴"的结构化描述 |
| 🧬 **药物发现** 💊 | 分子结构预测药性 |
| ☁️ **点云分类** | 3D 激光雷达点云识别 |
| 🦾 **姿态估计** | 人体骨架关节点识别 |
| 📝 **文本分类** | 文档引用网络节点分类 |
| 🛒 **推荐系统** | 用户-商品二分图推荐 |
| 🚗 **车联网** | 车辆节点通信与路径规划 |

## 一、📜 背景：为什么需要智能优化？

### 知识点 1：行为学派的起源

**Rodney Brooks（MIT, 1991）** — "Intelligence without Representation"

> 人工智能研究在"表征"问题上栽了跟头。如果通过感知和行动与真实世界交互，表征就不再是必须的。

Brooks 的核心理念：**智能不一定要有复杂的内部模型，行为本身就是智能** 🏃

### 知识点 2：行为学派的三个分支

```
┌─────────────────────────────────────────────┐
│  行为学派                                      │
├─────────────────────────────────────────────┤
│ ① 行为主义 (Actionism) → 智能优化              │
│ ② 进化主义 (Evolutionism) → 进化计算           │
│ ③ 控制论 (Cybernetics) → 控制与反馈             │
└─────────────────────────────────────────────┘
```

**控制论（Cybernetics）**：Norbert Wiener 1948 年提出，来自希腊词 kybernetēs（舵手），研究**通信与控制**的学科。

### 知识点 3：为什么要用智能优化？

**传统优化方法**（线性规划 LP、非线性规划 NLP、动态规划 DP）面对以下问题时会很吃力：
- 高维复杂问题（"维度灾难" 💥）
- 非凸优化（到处都是局部最优解 ⛰️）
- NP-hard 问题（计算量随规模指数爆炸 💣）

> 💡 **智能优化的优势**：不要求函数连续、可导；天然并行；能找到"足够好"的解

### 知识点 4：智能优化方法谱系

| 年份 | 方法 | 提出者 | 灵感来源 |
|:---- |:---- |:------ |:-------- |

| 1982 | **模拟退火（SA）** 🔥 | Kirkpatrick | 金属退火 |
| 1992 | **蚁群优化（ACO）** 🐜 | Dorigo | 蚂蚁觅食 |
| 1995 | **粒子群优化（PSO）** 🐦 | Kennedy & Eberhart | 鸟群飞行 |

## 二、🧬 进化计算 & 遗传算法（GA）（重点 ⭐）

### 知识点 5：进化计算的灵感来源

**生物进化给算法带来的启发：**
- **优胜劣汰**：适应度高的个体更容易存活和繁殖
- **遗传**：父母的特征传给后代（DNA 复制）
- **变异**：后代会发生随机变化，提供多样性
- **自然选择**：环境筛选出最适应生存的个体

### 知识点 6：GA 的基础概念——生物类比

| 术语 | 生物学含义 🧬 | GA 中的含义 |
|:---- |:------------ |:----------- |
| **染色体（Chromosome）** | DNA 链 | 一个候选解（编码后的） |
| **基因（Gene）** | DNA 片段 | 解的一个分量 |
| **等位基因（Allele）** | 基因的不同形式 | 编码取值（0 或 1） |
| **基因座（Locus）** | 基因在染色体上的位置 | 编码位置 |
| **表现型（Phenotype）** | 实际表现出来的性状 | 解码后的实际解 |
| **基因型（Genotype）** | 基因组成 | 编码后的解 |
| **种群（Population）** | 一群生物 | 一组候选解 |
| **适应度（Fitness）** | 生存能力 | 目标函数值（优劣程度） |

### 知识点 7：GA 的核心操作（三大算子）

#### 7.1 选择（Selection）🎯 — 优胜劣汰

根据适应度选择"优秀"的个体作为父母。

**常用方法——轮盘赌选择（Roulette Wheel Selection）：**
- 每个个体被选中的概率 ∝ 它的适应度
- 适应度越高 → 扇形越大 → 被选中的概率越大

```
个体:   v₁   v₂   v₃   v₄   v₅  ...  v₁₀
适应度:  F₁   F₂   F₃   F₄   F₅  ...  F₁₀
        ┌──────────────────────────────┐
        │ 🎡 轮盘赌：F 越大扇形越大     │
        └──────────────────────────────┘
        每个 v_i 被选中的概率 ≈ F_i / ΣF
```

#### 7.2 交叉（Crossover）✂️ — 遗传重组

把两个父代的染色体"剪开再拼接"，产生子代。

```
父代 A:  11001 | 01010        子代 A':  11001 | 10110
父代 B:  00110 | 10110        子代 B':  00110 | 01010
              ↑
          交叉点（随机选择）
```

- **交叉概率 p_c**（通常 ~0.25，即 25% 的概率发生交叉）

#### 7.3 变异（Mutation）🔄 — 基因突变

随机改变染色体上的某一位，引入新的基因多样性。

```
变异前:  1100101010
变异后:  1100111010
                ↑
          这一位翻转 0→1
```

- **变异概率 p_m**（通常很小，~0.01 或更低）

### 知识点 8：GA 的完整流程

```
Step 1：初始化种群（随机生成 N 个个体）
Step 2：计算每个个体的适应度
Step 3：判断是否满足终止条件 → 是则输出最优解
Step 4：选择（Selection）→ 选出父母
Step 5：交叉（Crossover）→ 生成子代
Step 6：变异（Mutation）→ 引入变化
Step 7：生成新一代种群 → 回到 Step 2
```

### 知识点 9：GA 实例——求函数最大值 🧮

**问题**：求 f(x₁, x₂) 在指定区间上的最大值

**编码方案：**
- 每个变量用二进制串表示
- 比如 x₁ 范围 [-3.0, 12.1]，精度要求 → 需要 18 位二进制
- x₂ 范围 [4.1, 5.8]，精度要求 → 需要 15 位二进制
- 一条染色体 = 18 + 15 = 33 位二进制串

**求解过程（代数演进）：**
```
第 1 代：  随机初始解，适应度较低
第 10 代： 开始聚集在较优区域
第 20 代： 接近最优解
第 40 代： f_min ≈ 6.655×10⁻⁴，已经很接近真实最优（0）
第 1000 代： 找到最优解 (11.63, 5.72)，f = 38.82
```

> 💡 GA 不是一步到位的，而是**一代一代逐步优化**的。

### 知识点 10：GA 的关键参数

| 参数 | 作用 | 典型值 |
|:---- |:---- |:------ |
| **种群大小** 👥 | 每代有多少个体 | 100~1000 |
| **交叉概率 p_c** ✂️ | 发生交叉的概率 | 0.25~0.8 |
| **变异概率 p_m** 🔄 | 每个基因变异的概率 | 0.001~0.1 |
| **终止条件** 🛑 | 什么时候停止 | 最大代数 / 收敛判断 |

## 三、🐝 群体智能（Swarm Intelligence）

### 知识点 11：群体智能的核心思想

**群体智能**：大量简单个体通过局部交互，涌现出全局智能行为 🐦

> 💡 **核心原则**：个体遵循简单规则，没有中央控制，集体表现出复杂的智能行为。

**三个基本规则：**
1. 🚶 **不碰撞** — 避免和邻居撞上
2. 🚀 **朝同一方向** — 和邻居保持大致方向一致
3. 🤝 **保持靠近** — 不要和群体离太远

### 知识点 12：群体智能的主要方法

| 方法 | 年份 | 灵感 |
|:---- |:---- |:---- |
| **蚁群优化（ACO）** 🐜 | 1992 (Dorigo) | 蚂蚁觅食路径 |
| **粒子群优化（PSO）** 🐦 | 1995 (Kennedy & Eberhart) | 鸟群飞行 |
| **人工蜂群（ABC）** 🐝 | 2005 | 蜜蜂采蜜 |
| **细菌觅食（BFO）** 🦠 | 2002 | 大肠杆菌觅食 |
| **萤火虫算法（FA）** ✨ | 2008 | 萤火虫发光 |
| **蜘蛛猴优化（SMO）** 🐒 | 2014 | 蜘蛛猴的社会行为 |

### 知识点 13：PSO（粒子群优化）—— 重点 ⭐

**PSO 模拟鸟群觅食 🐦：**

每只"鸟"（粒子）在搜索空间中飞行，同时记住两件事：
1. 🧑 **我自己找到过的最好位置**（个体历史最优 pBest）
2. 👥 **整个群体找到的最好位置**（全局最优 gBest）

**粒子的速度和位置更新公式：**
```
v_new = v_old + c₁·r₁·(pBest - x) + c₂·r₂·(gBest - x)
x_new = x_old + v_new
```

| 符号 | 含义 |
|:---- |:---- |
| v | 粒子速度（飞行的方向和快慢） |
| x | 粒子当前位置 |
| pBest 🧑 | 粒子自己历史最优 |
| gBest 👥 | 群体全局最优 |
| c₁, c₂ | 学习因子（自我认知 vs 社会认知） |
| r₁, r₂ | 随机数，增加探索性 |

> 💡 **PSO vs GA 的核心区别**：PSO 每个粒子始终记得自己的历史最优，GA 则是通过交叉变异产生新个体，**PSO 没有"灭绝"机制**。

### 知识点 14：ACO（蚁群优化）—— 重点 ⭐

**ACO 模拟蚂蚁觅食 🐜：**

蚂蚁在找食物时，会在走过的路径上留下**信息素（Pheromone）** 💫。信息素越多 → 该路径越"好" → 更多蚂蚁选择这条路径 → 信息素更多（正反馈 🔁）

**核心机制：**
- ✅ **正反馈**：好的路径信息素不断累积
- ✅ **多样性**：蚂蚁也会随机探索新路径
- ✅ **分布式**：没有中央控制，蚂蚁各自行动

**ACO 解决 TSP 问题（旅行商问题）🌍：**
- TSP：给定 n 个城市，找最短路径，每个城市恰好走一次
- 穷举法复杂度：O((n-1)!) — 当 n=30 时，需要 4.9×10²³ 年 💀
- ACO 能快速找到"足够好"的解

**ACO 的核心步骤：**
```
1. 初始化：每条边设置初始信息素
2. 每只蚂蚁构造路径（根据信息素 + 启发式信息选择下一城市）
3. 更新信息素：
   - 蒸发：所有路径信息素按比例减少（避免局部最优）
   - 增强：蚂蚁走过的路径增加信息素（好路径更突出）
4. 重复直到收敛
```

### 知识点 15：进化计算 vs 群体智能

| 对比 | 进化计算（GA） | 群体智能（PSO/ACO） |
|:---- |:------------- |:----------------- |
| **灵感来源** | 达尔文进化论 🧬 | 动物社会行为 🐝 |
| **基本单元** | 染色体（编码解） | 粒子/蚂蚁（直接位置） |
| **信息共享** | 交叉产生子代 | 个体 → 全局最优 |
| **记忆机制** | 无（子代覆盖父代） | 有（记住历史最优） |
| **适用场景** | 离散优化、组合优化 | 连续优化（PSO）、路径优化（ACO） |

## 三、📐 马尔可夫决策过程（MDP）（重点 ⭐）

### 知识点 5：MDP 的定义

**MDP = Markov Decision Process**，是 RL 问题的标准数学框架。

**MDP 用五元组表示**：⟨S, A, P, R, γ⟩
- **S** — 状态集合（所有可能的状态）
- **A** — 动作集合（所有可能的动作）
- **P** — 状态转移概率 P(s'|s, a)（做了动作 a 后到 s' 的概率）
- **R** — 奖励函数 R(s, a)（在 s 做 a 得到的即时奖励）
- **γ** — 折扣因子（0~1，未来的奖励折算到现在值多少）

### 知识点 6：马尔可夫性质（Markov Property）

**核心定义**：下一时刻的状态**只取决于当前状态和动作**，和之前的历史无关。

> P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, ...) = P(s_{t+1} | s_t, a_t)

就像下棋 ♟️：你不需要知道之前怎么走到这一步的，只看**当前棋盘**就能决定下一步怎么走。

### 知识点 7：完全可观测 vs 部分可观测

| 类型 | 说明 | 例子 |
|:---- |:---- |:---- |
| **完全可观测（MDP）** 👁️ | Agent 能看到环境的全部状态 | 围棋（棋盘一目了然） |
| **部分可观测（POMDP）** 👓 | Agent 只能看到部分状态 | 德州扑克（看不到对手的牌） |

> 💡 **POMDP** = Partially Observable Markov Decision Process。这种情况下 Agent 需要记住历史信息来推断真实状态。

## 四、💰 奖励、回报和价值函数

### 知识点 8：Reward（奖励）vs Return（回报）

**Reward（奖励）** r_t：每一步的即时反馈 📨
**Return（回报）** G_t：从当前步开始到结束的**累计折扣奖励** 🧮

```
G_t = r_{t+1} + γ·r_{t+2} + γ²·r_{t+3} + ...
```

**折扣因子 γ 的意义：**
- γ → 0：只看眼前利益（短视 👀）
- γ → 1：有远见，未来的奖励也很重要（深谋远虑 🧐）
- 一般取值 0.9~0.99

### 知识点 9：Value Function（价值函数）—— 核心概念 ⭐

**价值函数 = 预测未来能拿多少奖励** 🔮

**两种价值函数：**

| 函数 | 符号 | 含义 |
|:---- |:---- |:---- |
| **状态价值函数** 📊 | V(s) | 从状态 s 出发，按策略 π 行动，能拿到的期望回报 |
| **动作价值函数** 🎯 | Q(s, a) | 在状态 s 做了动作 a，之后按策略 π 行动，能拿到的期望回报 |

**公式：**
```
V_π(s) = E[G_t | S_t = s]                      = E[r₁ + γ·r₂ + γ²·r₃ + ... | S_t = s]
Q_π(s, a) = E[G_t | S_t = s, A_t = a]          = E[r₁ + γ·r₂ + γ²·r₃ + ... | S_t = s, A_t = a]
```

> 💡 **V(s) 和 Q(s,a) 的关系**：Q(s,a) 比 V(s) 多指定了第一步的动作。知道了 Q(s,a)，选最大 Q 值的动作就是最优策略。

### 知识点 10：最优价值函数与最优策略

RL 的最终目标：找到**最优策略 π\***，使得 V(s) 或 Q(s,a) 最大。

```
V*(s) = max_π V_π(s)     — 从状态 s 出发能得到的最大累计奖励
Q*(s,a) = max_π Q_π(s,a) — 在 s 做 a 能得到的最大累计奖励
```

**确定最优策略**：在每个状态 s，选 Q*(s,a) 最大的那个动作！
```
π*(s) = argmax_a Q*(s,a)
```

### 知识点 12：深度强化学习的关键突破 — DQN

**DQN（Deep Q-Network）** = Q-Learning + 深度学习 🧠

**为什么 DQN 是里程碑？**
- 用**神经网络**代替 Q 表格，处理高维状态（如原始像素）
- 两个关键 trick：

| Trick | 作用 |
|:------|:---- |
| **经验回放（Experience Replay）** 🔄 | 存下过去的经验，随机抽样训练，打破数据相关性 |
| **目标网络（Target Network）** 🎯 | 单独用一个网络算目标 Q 值，稳定训练 |

> 2013 年 DeepMind 用 DQN 在 Atari 2600 游戏上超越人类水平，是**深度强化学习**的开端 🚀

### 知识点 13：AlphaGo & AlphaZero

**AlphaGo（2016）：**
- 围棋状态空间 10¹⁷⁰，比宇宙原子总数还大 🌌
- 组合拳：**深度神经网络（策略网络+价值网络）+ 蒙特卡洛树搜索（MCTS）**
- 击败李世石（4:1），震动全球 🌍

**AlphaZero（2017）：**
- **完全从零自学**，不学人类棋谱，只靠自我对弈
- 同样的算法框架通吃围棋、象棋、将棋

## 六、🔑 RL 的核心要素 / 分类

### 知识点 15：RL 算法的三个核心要素

Agent 做决策需要三个关键组件：

```
┌──────────────────────────────────┐
│           RL Agent                │
│                                   │
│  ① Policy（策略） 📋               │
│     在状态 s 下，选择动作 a 的规则    │
│                                   │
│  ② Value Function（价值函数）💰     │
│     评估当前状态/动作有多好          │
│                                   │
│  ③ Model（模型）🌍                 │
│     对环境的理解（选配）             │
└──────────────────────────────────┘
```

### 知识点 16：Model-based vs Model-free

| 类型 | 说明 | 优点/缺点 |
|:---- |:---- |:--------- |
| **Model-based** 🧠 | Agent 试图"理解"环境，学习转移函数 P(s'\|s,a) 和奖励函数 R(s,a) | ✅ 更高效利用数据 ❌ 模型可能学错 |
| **Model-free** 🎮 | Agent 不学环境模型，直接学策略或价值函数 | ✅ 实现简单 ❌ 需要大量交互数据 |

### 知识点 17：Policy-based vs Value-based

| 类型 | 学什么 | 输出 | 适用场景 |
|:---- |:----- |:---- |:-------- |
| **Value-based** 📊 | 学价值函数 V(s) 或 Q(s,a) | 从 Q 值选最大动作 | 离散动作空间 |
| **Policy-based** 📋 | 直接学策略 π(a\|s) | 动作的概率分布 | 连续动作空间/随机策略 |
| **Actor-Critic** 🎭 | 两者都学 | Actor 做动作，Critic 评价 | 综合两者优势 |

## 二、🔍 逐知识点精讲（含真题回顾）

### 🥇 必考 1：谓词逻辑翻译与推理

> 📎 详细笔记：[symbolism.md → 知识点 8-11](symbolism.md)

**真题回顾**：选择题第 1 题，给出一组谓词逻辑公式判断正误。

**翻译口诀：**

```text
"所有 A 都是 B"  →  ∀x A(x) → B(x)    (用 →)
"有的 A 是 B"    →  ∃x A(x) ∧ B(x)    (用 ∧)
"没有 A 是 B"    →  ¬∃x A(x) ∧ B(x)  或 ∀x A(x) → ¬B(x)
"只有 A 才 B"    →  B → A
```

**⚠️ 经典陷阱：**
- ❌ `∀x A(x) ∧ B(x)` — 这表示"所有东西既是 A 又是 B"，完全不是"所有 A 都是 B"
- ❌ `∃x A(x) → B(x)` — 存在量词用 → 会出问题（空真问题）

**真题示例（2025 A 卷第 1 题）：** 选出正确的谓词逻辑公式 → 注意区分 ∀ 和 ∃ 的搭配。

---

### 🥇 必考 2：A\* 搜索算法

> 📎 详细笔记：[search.md → 知识点 10-12](search.md)

**真题回顾**：大题第 1 题（15 分，A\* 搜索），用两个启发式函数做图搜索。

**必须掌握的步骤：**

```text
A* 算法执行流程：
① 初始节点 S 放入 OPEN 表，f(S) = g(S) + h(S) = 0 + h(S)
② 从 OPEN 表取出 f(n) 最小的节点扩展
③ 对每个后继 n'：
   · g(n') = g(n) + c(n, n')
   · f(n') = g(n') + h(n')
   · 若 n' 不在 OPEN/CLOSED 中 → 加入 OPEN
   · 若 n' 已在 OPEN 中且新 f 更小 → 更新 f
   · 若 n' 已在 CLOSED 中且新 f 更小 → 重新放回 OPEN
④ 重复直到目标节点被扩展
```

**A\* vs Dijkstra vs 贪心：**

| 算法 | f(n) | 特点 |
| :--- | :--- | :--- |
| **Dijkstra** | f(n) = g(n) | 均匀向所有方向扩展 |
| **贪心最佳优先** | f(n) = h(n) | 直奔目标，但不一定最优 |
| **A\*** | f(n) = g(n) + h(n) | 两者平衡 |

**四个评估标准回顾：**

| 算法 | 完备性 | 最优性 | 数据结构 |
| :--- | :----: | :----: | :------- |
| DFS | ❌ | ❌ | 栈 |
| BFS | ✅ | ✅（等代价） | 队列 |
| UCS | ✅ | ✅ | 优先队列(g) |
| A\* | ✅（可采纳h） | ✅（可采纳h） | 优先队列(f=g+h) |

### 🥇 必考 3：CF 确定性因子模型

> 📎 详细笔记：[symbolism.md → 知识点 13](symbolism.md)

**真题回顾**：大题第 2 题（15 分），4 条规则合成计算 CF(H)。

**核心公式（必须记住）：**

```text
① AND 合取：CF(E) = min{CF(E₁), CF(E₂), ...}
② OR 析取： CF(E) = max{CF(E₁), CF(E₂), ...}
③ 规则执行：CF(H) = CF(H,E) × max{0, CF(E)}
④ 同一结论多规则合成：
    都 > 0:   CF₁₂ = CF₁ + CF₂ - CF₁·CF₂
    都 < 0:   CF₁₂ = CF₁ + CF₂ + CF₁·CF₂
    一正一负: CF₁₂ = (CF₁ + CF₂) / (1 - min(|CF₁|, |CF₂|))
```

**解题步骤：**
```text
Step 1: 从底层开始，计算中间结论的 CF
Step 2: 逐层向上推理
Step 3: 遇到同结论多规则 → 合成
```

### 🥇 必考 4：模糊逻辑与模糊集

> 📎 详细笔记：[symbolism.md → 知识点 14](symbolism.md)

**真题回顾**：大题第 3 题（15 分），给定论域和隶属度做模糊运算。

**核心运算：**

```text
① μ_A∩B(x) = min(μ_A(x), μ_B(x))          — 模糊交
② μ_A∪B(x) = max(μ_A(x), μ_B(x))          — 模糊并
③ μ_Ā(x) = 1 - μ_A(x)                      — 模糊补
④ μ_R(a,b) = min(μ_A(a), μ_B(b))          — 笛卡尔积
⑤ S = Q ∘ R:  μ_S(x,z) = max_y min{Q(x,y), R(y,z)}  — 模糊关系合成
```

### 🥇 必考 5：强化学习——折扣回报

> 📎 详细笔记：[reinforcement_learning.md → 知识点 8-9](reinforcement_learning.md)

**真题回顾**：多选第 8 题，FrozenLake 问题计算折扣回报。

**核心公式：**

```text
Gₜ = r₁ + γ·r₂ + γ²·r₃ + γ³·r₄ + ...

真题示例 FrozenLake：
   初始状态 → ... → Goal(+10), γ = 0.9
    中间每步奖励 = 0

如果走到 Goal 需要 5 步：
   G₀ = 0·1 + 0·0.9 + 0·0.9² + 0·0.9³ + 10·0.9⁴
      = 10 × 0.6561 = 6.561 ✅
```

### 🥇 必考 6：Transformer / Self-Attention

> 📎 详细笔记：[gan_att_gnn.md → 知识点 9-10](gan_att_gnn.md)

**真题回顾**：多选第 6 题，关于 Transformer 组件（QKV 等）。

**核心公式：**

```text
Self-Attention(Q,K,V) = softmax(Q·K^T / √d_k) · V

其中：
  Q = X × W_Q    (Query — 当前词想找什么)
  K = X × W_K    (Key — 每个词有什么信息)
  V = X × W_V    (Value — 每个词的实际内容)
  
除以 √d_k：防止点积过大导致 Softmax 梯度消失
```

**Multi-Head Attention：** 多个注意力头同时计算，每个头关注不同方面，最后拼接。

**1. 下列哪个是有效的谓词逻辑公式？** 🧪

A. ∀x Man(x) → Mortal(x)
B. ∀x (Mule(x) ∧ ¬Horse(x) ∧ ¬Donkey(x))
C. ∀x Person(x) → (Sleep(x) ∨ Eat(x))
D. ¬∀x Student(x) → Calculus(x)

**✅ 答案：A、C**

**解析：**
- A ✅：∀x (Man(x) → Mortal(x)) — "所有人都难免一死"，正确的蕴含式
- B ❌：∧ 连接的量词表示"所有东西都是骡子且不是马且不是驴"，不符合常识
- C ✅：∀x (Person(x) → (Sleep(x) ∨ Eat(x))) — "每个人都会睡觉或吃饭"
- D ❌：量词作用域不清晰，¬∀x 后面的括号缺失

**2. 关于 A\* 算法，下列说法正确的是？** ⭐

A. f(n) = g(n) + h(n)，g(n) 是启发式估计
B. 当 h(n) = 0 时，A\* 退化为贪心搜索
C. 可采纳的 h 保证 A\* 找到最优解
D. h(n) 越大，搜索效率越低

**✅ 答案：C**

**解析：**
- A ❌：g(n) 是**实际代价**，h(n) 才是启发式估计
- B ❌：h(n)=0 时退化为 **UCS**（Dijkstra），不是贪心搜索
- C ✅：可采纳性（h(n) ≤ h\*(n)）是 A\* 找到最优解的充分条件
- D ❌：h(n) 越大（在不超过 h\*(n) 的前提下），搜索效率越**高**（扩展更少节点）

> 📎 参考：[search.md → 知识点 11](search.md)

**3. 关于 CNN 的池化层，下列说法错误的是？** 🖼️

A. 可以降低特征图尺寸
B. 可以减少参数量
C. 可以增强平移不变性
D. 会让模型变得更深

**✅ 答案：D**

**解析：**
池化层作用是**降采样**（A✅）、**减少参数量**（B✅）、**增强平移不变性**（C✅），但不会改变网络深度（D❌）。网络深度由卷积层/全连接层的层数决定。

> 📎 参考：[cnn.md → 知识点 6](cnn.md)

**4. 在强化学习中，"Exploration vs Exploitation" 指的是？** 🎮

A. 探索新动作 vs 利用已知最优动作
B. 训练 vs 测试
C. 监督 vs 无监督
D. 策略网络 vs 价值网络

**✅ 答案：A**

**解析：** Exploration（探索）是尝试未做过的新动作，Exploitation（利用）是使用已知的最优策略。两者的平衡是 RL 的核心问题。

> 📎 参考：[reinforcement_learning.md → Exploration vs Exploitation](reinforcement_learning.md)

**5. 下列哪项不是 Hopfield 网络的特点？** 🔄

A. 全连接反馈结构
B. 能量函数单调下降
C. 通过 BPTT 训练
D. 可以用于联想记忆

**✅ 答案：C**

**解析：**
- A ✅：Hopfield 是单层全连接反馈网络
- B ✅：Hopfield 网络运行时能量函数单调递减，最终收敛到稳定态
- C ❌：**BPTT（Back Propagation Through Time）** 是 RNN 的训练方法，不是 Hopfield 的
- D ✅：DHNN 可用于联想记忆

> 📎 参考：[connectionism.md → Hopfield 网络](connectionism.md)

**6. 在 Transformer 中，Scaled Dot-Product Attention 除以 √d_k 的原因是？** 🤖

A. 防止梯度爆炸
B. 防止点积过大导致 Softmax 梯度消失
C. 加快计算速度
D. 增加模型容量

**✅ 答案：B**

**解析：** 当 d_k（Key 的维度）很大时，Q·Kᵀ 的点积结果会很大，导致 Softmax 函数进入饱和区，梯度极小（梯度消失）。除以 √d_k 将数值拉回到合适的范围。

> 📎 参考：[gan_att_gnn.md → 知识点 9.4](gan_att_gnn.md)

**7. 下列属于无信息搜索（盲搜）算法的有？** 🔍

A. DFS
B. BFS

D. UCS
E. Dijkstra

**✅ 答案：A、B、D、E**

**解析：** 无信息搜索 = 除了问题定义外不用任何额外信息

**8. 关于 CF 模型，下列说法正确的有？** 📊

A. CF 的取值范围是 [-1, 1]
B. AND 条件取各证据 CF 的最小值
C. OR 条件取各证据 CF 的最大值
D. CF > 0 表示证据支持结论
E. CF(H) = CF(H,E) × min{0, CF(E)}

**✅ 答案：A、B、C、D**

- A ✅：CF ∈ [-1, 1]
- B ✅：AND 取 min
- C ✅：OR 取 max
- D ✅：CF > 0 支持，CF < 0 反对
- E ❌：应该是 CF(H) = CF(H,E) × **max**{0, CF(E)}，不是 min

**9. 以下哪些是深度学习成功的关键因素？** 🚀

B. GPU 算力
C. 更好的算法（ReLU、Dropout）
D. 符号逻辑的突破

**✅ 答案：A、B、C**

**解析：** 深度学习成功的三要素：**大数据** + **大算力** + **好算法**（ReLU、Dropout等）。符号逻辑（D）的突破不是深度学习成功的原因。

> 📎 参考：[cnn.md → 知识点 2](cnn.md)

**10. 关于强化学习的 MDP，下列说法正确的有？** 🎯

A. MDP 满足马尔可夫性质
B. 折扣因子 γ 越接近 1 越"有远见"
C. Q(s,a) 表示在状态 s 做动作 a 的期望回报
D. Policy 定义了在每个状态做什么动作

**✅ 答案：A、B、C、D**

**解析：** 全部正确
- A ✅：马尔可夫性质 = 未来只取决于当前，与历史无关
- B ✅：γ→1 时未来的奖励几乎不打折扣（有远见）；γ→0 时只看眼前
- C ✅：Q(s,a) = E[Gₜ | Sₜ=s, Aₜ=a]
- D ✅：策略 π(a|s) 是在状态 s 下选择动作 a 的概率分布

**预测题 11：A\* 搜索 🧭（15分）**

**题目（见上文——此处省略重复）**

**✅ 参考答案：**

**(1) 判断可采纳性（3分）**

```text
h₁(n) 是否可采纳？
检查是否每个 n 都有 h₁(n) ≤ h*(n)（真实最短距离）

S 到 G 的真实最短距离：
  S→N4→N1→G = 1+2+2 = 5
  S→N3→G = 3+2 = 5
  S→N2→G = 2+3 = 5
  S→N4→N1→G = 5
真实 h*(S) = 5
h₁(S) = 6 > 5 → ❌ h₁ 不可采纳！

h₂(n) 是否可采纳？
假设经过计算所有 h₂(n) ≤ h*(n)
→ ✅ h₂ 可采纳（需要验证每个节点）
```

**(2) 用 h₁ 运行 A\* 的前 5 步（6分）**

```text
Step 0: OPEN = {S(g=0,h=6,f=6)}
Step 1: 扩展 S → 后继 N2(g=5,h=4,f=9), N3(g=3,h=3,f=6), N4(g=1,h=5,f=6)
         OPEN = {N3(f=6), N4(f=6), N2(f=9)}
Step 2: 扩展 N3(f=6) → 后继 G(g=3+2=5,h=0,f=5)
         OPEN = {G(f=5), N4(f=6), N2(f=9)}
Step 3: 扩展 G(f=5) → 找到目标 ✅
        总共只需 4 步扩展
```

**(3) h₁ vs h₂ 哪个更好？（6分）**

```text
h₂ 更好。因为 h₁ 不可采纳（h₁(S)=6 > h*(S)=5），
违背了 A* 最优性的前提。用 h₁ 时 A* 可能找不到最优解。

而 h₂ 可采纳，保证最优性。
并且 h₂ 更精确（更接近真实值），扩展节点数更少。
```

**预测题 12：CF 模型计算 📊（15分）**

**题目（见上文——此处省略重复）**

**✅ 参考答案：**

**(1) 计算 CF(E₃) 和 CF(E₅) （4分）**

```text
r₁: IF E₁ AND E₂ THEN E₃ (0.8)
CF(E₁ AND E₂) = min{CF(E₁), CF(E₂)} = min{0.7, 0.5} = 0.5
CF(E₃) = 0.8 × max{0, 0.5} = 0.4

r₂: IF E₃ OR E₄ THEN E₅ (0.9)
CF(E₃ OR E₄) = max{CF(E₃), CF(E₄)} = max{0.4, 0.4} = 0.4
CF(E₅) = 0.9 × max{0, 0.4} = 0.36
```

**(2) 分别计算 r₃ 和 r₄ 对 H 的 CF 值（6分）**

```text
r₃: IF E₅ THEN H (0.7)
CF₃(H) = 0.7 × max{0, 0.36} = 0.252

r₄: IF E₆ THEN H (0.85)
CF₄(H) = 0.85 × max{0, 0.8} = 0.68
```

**(3) 综合计算 CF(H) （5分）**

```text
都 > 0，用公式：CF₁₂ = CF₁ + CF₂ - CF₁·CF₂
CF(H) = 0.252 + 0.68 - 0.252 × 0.68
      = 0.932 - 0.17136
      = 0.76064 ≈ 0.761
```

**预测题 13：模糊推理 🌫️（15分）**

**题目（见上文——此处省略重复）**

**✅ 参考答案：**

**(1) A ∪ B 和 A ∩ B（5分）**

```text
A ∪ B (取 max):
  A∪B = {max(0,0)/100, max(0,0.1)/200, max(0.4,0.5)/500, max(0.7,0.8)/800, max(1.0,1.0)/1200}
      = {0/100, 0.1/200, 0.5/500, 0.8/800, 1.0/1200}

A ∩ B (取 min):
  A∩B = {min(0,0)/100, min(0,0.1)/200, min(0.4,0.5)/500, min(0.7,0.8)/800, min(1.0,1.0)/1200}
      = {0/100, 0/200, 0.4/500, 0.7/800, 1.0/1200}
```

**(2) ¬A（A 的补集）（4分）**

```text
¬A = {1-0/100, 1-0/200, 1-0.4/500, 1-0.7/800, 1-1.0/1200}
   = {1.0/100, 1.0/200, 0.6/500, 0.3/800, 0/1200}
```

**(3) 模糊关系 R = A → B（笛卡尔积）（6分）**

```text
R = A × B, μ_R(x,y) = min(μ_A(x), μ_B(y))

           B:  100   200   500   800   1200
A:   100  [  0    0     0     0     0   ]
     200  [  0    0     0     0     0   ]
     500  [  0    0.1   0.4   0.4   0.4 ]
     800  [  0    0.1   0.5   0.7   0.7 ]
    1200  [  0    0.1   0.5   0.8   1.0 ]

计算示例：μ_R(500, 200) = min(0.4, 0.1) = 0.1
         μ_R(1200, 1200) = min(1.0, 1.0) = 1.0
```
