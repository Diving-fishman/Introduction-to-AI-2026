## 一、📜 背景：为什么需要智能优化？

### 知识点 1：行为学派的起源

**Rodney Brooks（MIT, 1991）** — "Intelligence without Representation"

> 人工智能研究在"表征"问题上栽了跟头。如果通过感知和行动与真实世界交互，表征就不再是必须的。

Brooks 的核心理念：**智能不一定要有复杂的内部模型，行为本身就是智能** 🏃

### 知识点 2：行为学派的三个分支

```
┌─────────────────────────────────────────────┐
│  行为学派                                      │
├─────────────────────────────────────────────┤
│ ① 行为主义 (Actionism) → 智能优化              │
│ ② 进化主义 (Evolutionism) → 进化计算           │
│ ③ 控制论 (Cybernetics) → 控制与反馈             │
└─────────────────────────────────────────────┘
```

**控制论（Cybernetics）**：Norbert Wiener 1948 年提出，来自希腊词 kybernetēs（舵手），研究**通信与控制**的学科。

### 知识点 3：为什么要用智能优化？

**传统优化方法**（线性规划 LP、非线性规划 NLP、动态规划 DP）面对以下问题时会很吃力：
- 高维复杂问题（"维度灾难" 💥）
- 非凸优化（到处都是局部最优解 ⛰️）
- NP-hard 问题（计算量随规模指数爆炸 💣）

> 💡 **智能优化的优势**：不要求函数连续、可导；天然并行；能找到"足够好"的解

### 知识点 4：智能优化方法谱系

## 二、🧬 进化计算 & 遗传算法（GA）（重点 ⭐）

### 知识点 5：进化计算的灵感来源

**生物进化给算法带来的启发：**
- **优胜劣汰**：适应度高的个体更容易存活和繁殖
- **遗传**：父母的特征传给后代（DNA 复制）
- **变异**：后代会发生随机变化，提供多样性
- **自然选择**：环境筛选出最适应生存的个体

### 知识点 6：GA 的基础概念——生物类比

| 术语 | 生物学含义 🧬 | GA 中的含义 |
|:---- |:------------ |:----------- |
| **染色体（Chromosome）** | DNA 链 | 一个候选解（编码后的） |
| **基因（Gene）** | DNA 片段 | 解的一个分量 |
| **等位基因（Allele）** | 基因的不同形式 | 编码取值（0 或 1） |
| **基因座（Locus）** | 基因在染色体上的位置 | 编码位置 |
| **表现型（Phenotype）** | 实际表现出来的性状 | 解码后的实际解 |
| **基因型（Genotype）** | 基因组成 | 编码后的解 |
| **种群（Population）** | 一群生物 | 一组候选解 |
| **适应度（Fitness）** | 生存能力 | 目标函数值（优劣程度） |

### 知识点 7：GA 的核心操作（三大算子）

#### 7.1 选择（Selection）🎯 — 优胜劣汰

根据适应度选择"优秀"的个体作为父母。

**常用方法——轮盘赌选择（Roulette Wheel Selection）：**
- 每个个体被选中的概率 ∝ 它的适应度
- 适应度越高 → 扇形越大 → 被选中的概率越大

```
个体:   v₁   v₂   v₃   v₄   v₅  ...  v₁₀
适应度:  F₁   F₂   F₃   F₄   F₅  ...  F₁₀
        ┌──────────────────────────────┐
        │ 🎡 轮盘赌：F 越大扇形越大     │
        └──────────────────────────────┘
        每个 v_i 被选中的概率 ≈ F_i / ΣF
```

#### 7.2 交叉（Crossover）✂️ — 遗传重组

把两个父代的染色体"剪开再拼接"，产生子代。

```
父代 A:  11001 | 01010        子代 A':  11001 | 10110
父代 B:  00110 | 10110        子代 B':  00110 | 01010
              ↑
          交叉点（随机选择）
```

- **交叉概率 p_c**（通常 ~0.25，即 25% 的概率发生交叉）

#### 7.3 变异（Mutation）🔄 — 基因突变

随机改变染色体上的某一位，引入新的基因多样性。

```
变异前:  1100101010
变异后:  1100111010
                ↑
          这一位翻转 0→1
```

- **变异概率 p_m**（通常很小，~0.01 或更低）

### 知识点 8：GA 的完整流程

```
Step 1：初始化种群（随机生成 N 个个体）
Step 2：计算每个个体的适应度
Step 3：判断是否满足终止条件 → 是则输出最优解
Step 4：选择（Selection）→ 选出父母
Step 5：交叉（Crossover）→ 生成子代
Step 6：变异（Mutation）→ 引入变化
Step 7：生成新一代种群 → 回到 Step 2
```

### 知识点 9：GA 实例——求函数最大值 🧮

**问题**：求 f(x₁, x₂) 在指定区间上的最大值

**编码方案：**
- 每个变量用二进制串表示
- 比如 x₁ 范围 [-3.0, 12.1]，精度要求 → 需要 18 位二进制
- x₂ 范围 [4.1, 5.8]，精度要求 → 需要 15 位二进制
- 一条染色体 = 18 + 15 = 33 位二进制串

**求解过程（代数演进）：**
```
第 1 代：  随机初始解，适应度较低
第 10 代： 开始聚集在较优区域
第 20 代： 接近最优解
第 40 代： f_min ≈ 6.655×10⁻⁴，已经很接近真实最优（0）
第 1000 代： 找到最优解 (11.63, 5.72)，f = 38.82
```

> 💡 GA 不是一步到位的，而是**一代一代逐步优化**的。

### 知识点 10：GA 的关键参数

| 参数 | 作用 | 典型值 |
|:---- |:---- |:------ |
| **种群大小** 👥 | 每代有多少个体 | 100~1000 |
| **交叉概率 p_c** ✂️ | 发生交叉的概率 | 0.25~0.8 |
| **变异概率 p_m** 🔄 | 每个基因变异的概率 | 0.001~0.1 |
| **终止条件** 🛑 | 什么时候停止 | 最大代数 / 收敛判断 |

## 三、🐝 群体智能（Swarm Intelligence）

### 知识点 11：群体智能的核心思想

**群体智能**：大量简单个体通过局部交互，涌现出全局智能行为 🐦

> 💡 **核心原则**：个体遵循简单规则，没有中央控制，集体表现出复杂的智能行为。

**三个基本规则：**
1. 🚶 **不碰撞** — 避免和邻居撞上
2. 🚀 **朝同一方向** — 和邻居保持大致方向一致
3. 🤝 **保持靠近** — 不要和群体离太远

### 知识点 12：群体智能的主要方法

| 方法 | 年份 | 灵感 |
|:---- |:---- |:---- |
| **蚁群优化（ACO）** 🐜 | 1992 (Dorigo) | 蚂蚁觅食路径 |
| **粒子群优化（PSO）** 🐦 | 1995 (Kennedy & Eberhart) | 鸟群飞行 |
| **人工蜂群（ABC）** 🐝 | 2005 | 蜜蜂采蜜 |
| **细菌觅食（BFO）** 🦠 | 2002 | 大肠杆菌觅食 |
| **萤火虫算法（FA）** ✨ | 2008 | 萤火虫发光 |
| **蜘蛛猴优化（SMO）** 🐒 | 2014 | 蜘蛛猴的社会行为 |

### 知识点 13：PSO（粒子群优化）—— 重点 ⭐

**PSO 模拟鸟群觅食 🐦：**

每只"鸟"（粒子）在搜索空间中飞行，同时记住两件事：
1. 🧑 **我自己找到过的最好位置**（个体历史最优 pBest）
2. 👥 **整个群体找到的最好位置**（全局最优 gBest）

**粒子的速度和位置更新公式：**
```
v_new = v_old + c₁·r₁·(pBest - x) + c₂·r₂·(gBest - x)
x_new = x_old + v_new
```

| 符号 | 含义 |
|:---- |:---- |
| v | 粒子速度（飞行的方向和快慢） |
| x | 粒子当前位置 |
| pBest 🧑 | 粒子自己历史最优 |
| gBest 👥 | 群体全局最优 |
| c₁, c₂ | 学习因子（自我认知 vs 社会认知） |
| r₁, r₂ | 随机数，增加探索性 |

> 💡 **PSO vs GA 的核心区别**：PSO 每个粒子始终记得自己的历史最优，GA 则是通过交叉变异产生新个体，**PSO 没有"灭绝"机制**。

### 知识点 14：ACO（蚁群优化）—— 重点 ⭐

**ACO 模拟蚂蚁觅食 🐜：**

蚂蚁在找食物时，会在走过的路径上留下**信息素（Pheromone）** 💫。信息素越多 → 该路径越"好" → 更多蚂蚁选择这条路径 → 信息素更多（正反馈 🔁）

**核心机制：**
- ✅ **正反馈**：好的路径信息素不断累积
- ✅ **多样性**：蚂蚁也会随机探索新路径
- ✅ **分布式**：没有中央控制，蚂蚁各自行动

**ACO 解决 TSP 问题（旅行商问题）🌍：**
- TSP：给定 n 个城市，找最短路径，每个城市恰好走一次
- 穷举法复杂度：O((n-1)!) — 当 n=30 时，需要 4.9×10²³ 年 💀
- ACO 能快速找到"足够好"的解

**ACO 的核心步骤：**
```
1. 初始化：每条边设置初始信息素
2. 每只蚂蚁构造路径（根据信息素 + 启发式信息选择下一城市）
3. 更新信息素：
   - 蒸发：所有路径信息素按比例减少（避免局部最优）
   - 增强：蚂蚁走过的路径增加信息素（好路径更突出）
4. 重复直到收敛
```

### 知识点 15：进化计算 vs 群体智能

| 对比 | 进化计算（GA） | 群体智能（PSO/ACO） |
|:---- |:------------- |:----------------- |
| **灵感来源** | 达尔文进化论 🧬 | 动物社会行为 🐝 |
| **基本单元** | 染色体（编码解） | 粒子/蚂蚁（直接位置） |
| **信息共享** | 交叉产生子代 | 个体 → 全局最优 |
| **记忆机制** | 无（子代覆盖父代） | 有（记住历史最优） |
| **适用场景** | 离散优化、组合优化 | 连续优化（PSO）、路径优化（ACO） |
