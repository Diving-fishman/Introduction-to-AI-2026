## 一、📜 什么是符号学派？

### 知识点 1：符号学派的核心理念

**智能的本质 = 对符号进行操作和推理** 🧮

```text
符号主义的信念：
  ✅ 知识可以用"符号"来表示（如"猫"、"动物"）
  ✅ 推理就是按照规则操作这些符号
  ✅ 只要知识库足够大、推理规则足够强，就能实现智能

典型代表：
  · "知识就是力量" —— 专家系统时代的口号 💪
  · "物理符号系统假说" —— Newell & Simon
```

**符号学派的六步框架：**
```text
① 问题建模    ② 定义符号    ③ 构建规则
④ 推理求解    ⑤ 解释结果    ⑥ 知识更新
```

### 知识点 2：符号学派的优势与局限

| 优势 ✅ | 局限 ❌ |
| :------ | :------ |
| 推理过程**可解释**（能说清为什么） | 知识获取困难（人工输入规则成本高） |
| 可以处理**小样本**问题 | 难以处理**不确定性** |
| 知识可以**积累和复用** | 规则的"组合爆炸" 💥 |
| 逻辑严密，**结果确定** | 面对开放世界非常脆弱 |

## 二、🔗 知识表示（Knowledge Representation）

### 知识点 3：知识表示的基本概念

**知识表示 = 把现实世界的信息编码成计算机可以理解和操作的形式**

```text
知识表示的四种方法：
① 逻辑表示法（谓词逻辑） 📋
② 产生式表示法（IF-THEN 规则） ➡️
③ 框架表示法（对象-属性） 🖼️
④ 语义网络 / 知识图谱 🕸️
```

### 知识点 4：知识表示的理想特性

| 特性 | 含义 |
| :---- | :---- |
| **充分表示** 🎯 | 能表达所有需要表示的知识 |
| **充分推理** 🧠 | 支持对表示的知识进行推理 |
| **充分高效** ⚡ | 存储和操作效率高 |
| **充分获取** 📥 | 方便知识获取和更新 |

### 知识点 5：产生式系统（Production System）—— IF-THEN 规则

**最经典的符号主义知识表示方法**：

```text
IF <条件> THEN <结论/动作>

例子：
  IF 有发热 AND 有咳嗽 THEN 可能是感冒
  IF 是动物 AND 会飞 AND 有羽毛 THEN 是鸟
```

**产生式系统的三要素：**

| 要素 | 作用 | 比喻 📚 |
| :---- | :---- | :------ |
| **规则库** 📋 | 存储所有 IF-THEN 规则 | 法律条文汇编 |
| **综合数据库** 📦 | 存储当前已知的事实 | 案件事实记录 |
| **推理机** ⚙️ | 选择并执行规则 | 法官判案 |

**推理机的工作方式：**
```text
① 匹配：找所有条件满足当前事实的规则
② 冲突消解：如果多条规则同时满足，选哪条？
   - 优先级最高
   - 条件最具体
   - 最近用过的
③ 执行：应用选中的规则，更新事实库
④ 重复直到得出结论
```

**一个专家系统的推理过程：**
```text
事实：感冒了、发热、咳嗽
  ↓
匹配规则：IF 发热 AND 咳嗽 THEN 可能是流感
  ↓
结论：可能是流感 → 建议去医院检查
```

## 三、🧮 逻辑（Logic）—— 符号推理的基础

### 知识点 6：什么是逻辑？

**逻辑 = 研究正确推理形式的科学**

```text
"1+1=2" vs "1+1=10" 
两种说法都对，但基于不同的"世界"（十进制 vs 二进制）
逻辑就是告诉我们：在给定前提下，什么结论是必然成立的。
```

### 知识点 7：三种推理方式

| 推理 | 方向 | 特点 | 例子 |
| :---- | :---- | :---- | :---- |
| **演绎推理** 🔽 | 一般 → 个别 | 结论必然成立 | 所有人会死，苏格拉底是人 → 苏格拉底会死 |
| **归纳推理** 🔼 | 个别 → 一般 | 结论不一定成立 | 看到 100 只天鹅都是白的 → 天鹅都是白的？❌ |
| **默认推理** ➡️ | 缺省推理 | 没有反例就默认成立 | 鸟会飞 → 但企鹅不会飞（例外） |

### 知识点 8：命题逻辑（Propositional Logic）

**命题 = 可以判断真假的陈述句。**

```text
  P: "今天是晴天"  → 真或假
  Q: "我去游泳"    → 真或假
  
  逻辑连接词：
  ¬P   (非)    "今天不是晴天"
  P∧Q  (与)    "今天是晴天 且 我去游泳"
  P∨Q  (或)    "今天是晴天 或 我去游泳"
  P→Q  (蕴含)  "如果今天是晴天，我就去游泳"
  P↔Q  (等价)  "今天是晴天当且仅当我去游泳"
```

**真值表：**
```text
P    Q    ¬P    P∧Q    P∨Q    P→Q    P↔Q
T    T    F     T      T      T      T
T    F    F     F      T      F      F
F    T    T     F      T      T      F
F    F    T     F      F      T      T
```

> ⚠️ **P→Q 的重要规则**：只有当 P 真且 Q 假时，P→Q 为假。P 假时 P→Q 恒真！

### 知识点 9：谓词逻辑（Predicate Logic / FOL）—— 重点 ⭐

**命题逻辑的局限**：无法表达"所有人都会死"这种包含**量词**和**变量**的陈述。
**谓词逻辑 = 命题逻辑的升级版** 🚀

**核心元素：**

```text
① 谓词（Predicate）：表示关系或属性
   "x > y" → Greater(x, y)
   "x是人" → Human(x)
   "x喜欢y" → Like(x, y)

② 量词（Quantifier）：
   ∀x（全称量词）：所有 x 都...
     ∀x Human(x) → Mortal(x)   "所有人都难免一死"
   ∃x（存在量词）：存在某个 x 使得...
     ∃x Human(x) ∧ Smart(x)   "存在聪明的人"

③ 个体 / 常量：
   Socrates, Lisi, 华科...

④ 函数：
   son(x)  "x的儿子"
   father(x) "x的父亲"
```

**用谓词逻辑表达知识：**

```text
"所有学生都学习过人工智能课程"
∀x Student(x) → Study(x, AI)

"有的学生喜欢所有教授"
∃x Student(x) ∧ ∀y Professor(y) → Like(x, y)

"没有免费的午餐"
¬∃x Free(x) ∧ Lunch(x)
```

### 知识点 10：谓词逻辑推理示例

**经典三段论在谓词逻辑中的表达：**

```text
前提1：所有人都难免一死
        ∀x Human(x) → Mortal(x)
前提2：苏格拉底是人
        Human(Socrates)
结论：苏格拉底难免一死
        Mortal(Socrates)

推理过程：
① ∀x Human(x) → Mortal(x)
   用 Socrates 替换 x → Human(Socrates) → Mortal(Socrates)
② Human(Socrates) 已知为真
③ 所以 Mortal(Socrates) 为真 ✅
```

**更复杂的例子——"王喜欢聪明的东西"：**

```text
已知：
① ∀x Easy(x) → Like(Wang, x)    "王喜欢容易的科目"
② ∀x C(x) → Easy(x)              "所有计算机课都是容易的"
③ C(AI)                            "AI 是计算机课"

证明：Like(Wang, AI)  "王喜欢AI"

推理：
④ C(AI) → Easy(AI)              从②代入 x=AI
⑤ Easy(AI)                       从③④用假言推理
⑥ Easy(AI) → Like(Wang, AI)     从①代入 x=AI
⑦ Like(Wang, AI) ✅              从⑤⑥用假言推理
```

### 知识点 11：谓词逻辑的翻译练习（必做题 ⭐）

| 中文 | 谓词逻辑表达 |
| :---- | :---- |
| 人人都有母亲 | ∀x ∃y Mother(y, x) |
| 不存在既穿红又穿绿的人 | ¬∃x Red(x) ∧ Green(x) |
| 每个正整数都有唯一一个后继 | ∀x ∃y Succ(y, x) ... |
| 所有学生要么勤奋要么聪明 | ∀x S(x) → H(x) ∨ C(x) |

## 四、⚖️ 不确定性推理

### 知识点 12：为什么需要处理不确定性？

现实世界中的知识往往**不是确定的**：

```text
"发烧 + 咳嗽 → 可能是感冒"  （不是 100% 确定）
"乌云密布 → 可能要下雨"     （不是 100% 确定）
```

**不确定性的来源：**
- 知识本身不完整 🕳️
- 证据不充分 ❓
- 规则本身是统计性的 📊

### 知识点 13：确定性因子模型（C-F Model）—— 重点 ⭐

**MYCIN 医疗专家系统（1975, Shortliffe）** 使用的方法。

#### 基本概念

**CF（Certainty Factor，确定性因子）**：表示结论的可信程度，取值范围 [-1, 1]

```text
CF(H,E) = 可信度
├─ CF > 0：证据支持结论（越大越肯定）
├─ CF < 0：证据反对结论（越小越否定）
└─ CF = 0：证据与结论无关

例子：
  IF 发热 AND 咳嗽 THEN 感冒 (CF = 0.7)
  IF 白细胞升高 THEN 细菌感染 (CF = 0.8)
```

#### C-F 模型的计算规则

**① 证据的合取与析取：**
```text
合取（AND）：CF(E) = min{CF(E₁), CF(E₂), ...}
析取（OR）： CF(E) = max{CF(E₁), CF(E₂), ...}

例子：IF E₁ AND E₂ THEN H
CF(E₁) = 0.6, CF(E₂) = 0.8
CF(E) = min(0.6, 0.8) = 0.6
```

**② 规则的执行：**
```text
CF(H) = CF(H,E) × max{0, CF(E)}

如果 CF(E) < 0：规则不触发，CF(H) = 0
如果 CF(E) = 1：CF(H) = CF(H,E)（规则的可信度完全传递）

例子：IF E THEN H (0.7), CF(E) = 0.6
CF(H) = 0.7 × 0.6 = 0.42
```

**③ 多条规则的综合（同结论有多个证据）：**

当两条规则指向同一结论 H 时的合成公式：

```text
CF₁₂(H) = CF₁(H) + CF₂(H) - CF₁(H)·CF₂(H)     当两者都 > 0
CF₁₂(H) = CF₁(H) + CF₂(H) + CF₁(H)·CF₂(H)     当两者都 < 0
CF₁₂(H) = (CF₁(H) + CF₂(H)) / (1 - min(|CF₁|, |CF₂|))  当一正一负
```

**C-F 综合计算示例：**

```text
规则：
  r₁: IF E₁ THEN H (0.8)
  r₂: IF E₂ THEN H (0.9)
  r₃: IF E₁ AND E₃ THEN E₂ (0.7)

证据：
  CF(E₁) = 0.6, CF(E₃) = 0.8

计算：
  ① r₁: CF₁(H) = 0.8 × max{0, 0.6} = 0.48
  ② r₃: CF(E₂) = 0.7 × min{0.6, 0.8} = 0.42
  ③ r₂: CF₂(H) = 0.9 × max{0, 0.42} = 0.378
  ④ 综合 CF(H) = 0.48 + 0.378 - 0.48×0.378 = 0.677 ✅
```

### 知识点 14：模糊逻辑（Fuzzy Logic）🌫️

**1965 年 L.A. Zadeh 提出**，解决"非此即彼"的二元逻辑无法处理的问题。

#### 模糊集合

**核心思想**：元素可以**部分地**属于一个集合，隶属度 ∈ [0, 1]。

```text
传统集合：一个人要么"高"要么"不高"（0 或 1） ❌
模糊集合：一个人"高"的程度是 0.8 ✅

{"高"} = {0.95/180cm, 0.90/175cm, 0.85/170cm, ...}
```

**隶属函数 μ_A(x)** 表示元素 x 对集合 A 的隶属程度。

#### 模糊逻辑的基本运算

| 运算 | 公式 | 类似 |
| :---- | :---- | :---- |
| 交集（AND） | μ_A∩B(x) = min(μ_A(x), μ_B(x)) | 取小的 |
| 并集（OR） | μ_A∪B(x) = max(μ_A(x), μ_B(x)) | 取大的 |
| 补集（NOT） | μ_Ā(x) = 1 - μ_A(x) | 取反 |

#### 模糊推理示例

```text
规则：IF 温度是"高" THEN 风扇速度是"大"
输入：温度 = 35°C，对"高"的隶属度 = 0.9
输出：风扇速度调到 0.9 程度的大

这就像空调的变频控制——不是简单的"开/关"，而是"开多少" 🌡️
```

## 五、🕸️ 语义网络与知识图谱

### 知识点 15：语义网络（Semantic Network）

**1968 年提出**，用图结构表示知识。

```text
语义网络的基本元素：
  · 节点 🟢 = 概念、对象、事件
  · 边 🔗 = 关系（ISA、HAS-PART、OWNS...）

例子：
       XIAOYAN ——ISA——→ SWALLOW ——ISA——→ BIRD
         │                                    │
         └——OWNS——→ NEST-1 ——ISA——→ NEST     │
                                               │
                                         HAS-PART
                                               │
                                             WINGS
```

**语义网络的推理（继承机制）：**
```text
XIAOYAN ISA SWALLOW ISA BIRD
BIRD HAS-PART WINGS

→ 推理：XIAOYAN HAS-PART WINGS ✅
（子类自动继承父类的属性）
```

### 知识点 16：知识图谱（Knowledge Graph）—— 重点 ⭐

**知识图谱 = 语义网络的大规模升级版** 🚀

#### 核心概念

| 元素 | 含义 | 例子 |
| :---- | :---- | :---- |
| **实体（Entity）** 🧑 | 现实世界中的对象 | 特朗普、华中科技大学 |
| **概念（Concept）** 📋 | 实体的类别 | 人、大学、国家 |
| **关系（Relation）** 🔗 | 实体之间的联系 | spouse_of、located_in |
| **属性（Property）** 📊 | 实体的特征 | 年龄、身高、颜色 |

#### 著名知识图谱项目

| 项目 | 规模 | 特点 |
| :---- | :---- | :---- |
| **Cyc** 🏛️ | 70 万概念、630 万断言 | 手工构建的"常识百科全书" |
| **WordNet** 📖 | 英语词汇的语义网络 | 同义词集（synset） |
| **ConceptNet** 🌐 | 众包构建的常识知识库 | 基于 Open Mind Common Sense |
| **Freebase/Wikidata** 📚 | 49,915,906 实体 | 维基百科的结构化版本 |
| **Dbpedia** 🌍 | 127 万实体 | 从维基百科提取的结构化数据 |
| **Google KG** 🔍 | 5.7 亿实体、180 亿关系 | 2012 年推出，用于增强搜索 |
| **Probase** 🧮 | 1200 万概念 | 微软基于 Bing 的 IsA 关系提取 |

#### 知识图谱的构建流程

```text
① 实体识别   ② 关系抽取   ③ 实体链接
  ④ 知识融合   ⑤ 质量评估   ⑥ 知识更新
```

#### 知识图谱补全（Knowledge Graph Completion）

```text
知识图谱通常是不完整的，需要预测缺失的关系：

？→(妻子)→ 特朗普          → 梅拉尼娅
奥巴马 →(配偶)→ ？          → 米歇尔
中国 →(首都)→ ？            → 北京

三种预测：
· (h, r, ?) — 尾实体预测："中国的首都是什么？"
· (?, r, t) — 头实体预测："谁的首都是北京？"
· (h, ?, t) — 关系预测："中国和北京之间是什么关系？"
```

### 知识点 17：谓词逻辑和 C-F 模型的综合应用

#### 谓词逻辑推理题
**前提：**
1. 所有学生都会通过考试
2. 张三是学生
3. 所有通过考试的人都会毕业

**证明：张三会毕业**

```text
① ∀x S(x) → P(x)
② S(Zhang)
③ ∀x P(x) → G(x)

④ S(Zhang) → P(Zhang)   ①代入
⑤ P(Zhang)               ②④假言推理
⑥ P(Zhang) → G(Zhang)   ③代入
⑦ G(Zhang) ✅            ⑤⑥假言推理
```

#### C-F 模型计算题
```text
规则：
  r₁: IF E₁ AND E₂ THEN E₃ (0.9)
  r₂: IF E₃ OR E₄ THEN E₅ (0.8)
  r₃: IF E₅ THEN H (0.7)
  r₄: IF E₆ THEN H (0.9)

证据：
  CF(E₁) = 0.6, CF(E₂) = 0.5, CF(E₄) = 0.4, CF(E₆) = 0.8

求 CF(H)
```
